# 썸네일 문구 생성

## Overview

소식글 텍스트를 입력받아 구글 비즈니스 프로필 썸네일용 문구를 생성한다.
언어별 2줄 구성: 설명줄(서브문구) + 제목줄(메인문구)

## 실행 순서

1. 매장명으로 해당 `store-info.md` 읽기
2. `작업 언어` 필드 확인
3. 소식글 핵심 메시지 추출
4. 각 언어별 썸네일 문구 생성 후 한번에 출력

## 출력 형식

언어별 2줄, 모든 언어 묶어서 한번에 출력. 전체 1,300자 이하.

```
[언어명]
설명줄: (맥락/상황 설명 — 상대적으로 길어도 됨, 작은 폰트)
제목줄: (핵심 메시지 — 짧고 임팩트 있게, 큰 폰트)
```

## 언어별 톤

| 언어 | 톤 기준 |
|------|---------|
| 한국어 | 매장 store-info.md 톤 기준 |
| 영어 | 캐주얼 구어체 ("You'll love", "Worth the visit") |
| 일본어 | 정중 구어체 (~ます/です), "ぜひ一度", "リピ確定" |
| 중국어(간체) | 샤오홍슈/다중디엔핑 톤, "值得一去", "强烈推荐" |
| 중국어(번체) | 번체 기준, 자연스러운 추천체 |

## 주의사항

- 번역이 아니라 해당 언어권 구어체로 재작성
- 접속사로 자연스러운 흐름. 한 문단 시각적 기준 최대 2~3줄
- 소식글 본문 생성 금지 — 썸네일 문구 추출 전용
- 제목줄에 반드시 서비스명 또는 메뉴명 포함 — 추상적인 문구만으로 끝내지 않음
- 제목줄에 매장명(업장명) 넣지 않음 — 시술/서비스명만 깔끔하게
- 제목줄 카피라이팅 원칙: 메뉴명은 그대로 쓰되 최대한 간결하게. 수식어 남발 금지. 복수 메뉴 나열 시 &로 연결. 예) 단발 레이어드컷 & 보더펌 / Layered Bob & Border Perm
- 설명줄 길이 기준: 한국어 10자 이내, 영어 20자 이내 (45px 기준 1줄 이하). 핵심 키워드만. "~에서", "~에 맞춰", "~으로" 같은 연결어 붙이지 마라. 장황한 설명 금지.
- 특수기호 사용 금지 — 대시(-), 가운뎃점(·), 슬래시(/), 말줄임표(…) 등 이모지 외 모든 특수기호 제외. 단, 제목줄에서 메뉴/서비스 나열 시 &는 허용
- 의료 매장 (치과, 의원): 구글작업 CLAUDE.md 의료법 광고 규제 섹션 준수
  - 최상급 표현, 효과 보장, 전후 비교 표현 금지
