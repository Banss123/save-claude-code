# 리뷰 원고 검수

## Overview

작성된 리뷰 답변 원고를 받아 언어별 오류 패턴과 키워드 정합성을 검수한다.
생성이 아닌 검수 전용. 복수 원고 한번에 처리 가능.

## 실행 순서

1. 매장명으로 `store-info.md` 읽기 — 지점별 지정 키워드 확인
2. 원고 언어 감지
3. Step 1 → Step 2 → Step 3 순서대로 전체 원고 검수
4. 문제 항목을 심각도 순으로 정리해서 출력
5. 수정 원고 제공 (문제 항목만 재작성)

---

## Step 1 — 키워드 정합성 (최우선)

모든 언어 공통. 이 단계 오류가 가장 치명적.

### 체크 항목

- [ ] 이 매장 지정 키워드가 실제로 삽입되어 있는가
- [ ] 다른 지점 키워드가 혼용되지 않았는가 (지점이 여러 개인 매장 필수)
- [ ] 동일 원고 안에 키워드가 2회 이상 등장하는가 → 두 번째부터 유의어/지시어로 교체
- [ ] **⚠️ 외국어 답변 본문에 한국어(가-힣) 침투 없는가** — 게시될 답변 본문(서론·본론·결론)을 정규식 `[가-힣]`으로 훑어 1자라도 매칭되면 즉시 지적. 키워드 옆 괄호 한국어 병기(`ホンデ 皮膚科(홍대 피부과)`, `english speaking dermatologist(영어 가능 피부과)` 등)는 [한국어 해석] 블록과 [사용 키워드] 블록에서만 허용. 본문 침투는 일본어/영어/중국어 고객에게 그대로 노출되는 치명적 오류

### 한국어 침투 오류 예시

| 잘못된 예 (본문) | 수정 예 (본문) | 한국어 병기 위치 |
|---|---|---|
| `ホンデ 皮膚科(홍대 피부과)をお探しの方` | `ホンデ 皮膚科をお探しの方` | 하단 [사용 키워드] 블록 |
| `looking up english speaking dermatologist(영어 피부과)` | `looking up english speaking dermatologist` | 하단 [사용 키워드] 블록 |
| `下次來弘大皮膚科(홍대 피부과)時` | `下次來弘大皮膚科時` | 하단 [사용 키워드] 블록 |

### 오류 예시

| 오류 유형 | 잘못된 예 | 수정 예 |
|---------|---------|--------|
| 타 지점 키워드 혼용 | 이태원점 원고에 `restaurants near me` 사용 (압구정 키워드) | `Itaewon restaurants`로 교체 |
| 키워드 누락 | 원고 전체에 지정 키워드 없음 | 검색 맥락 문장으로 문두에 자연 삽입 |
| 키워드 중복 | 같은 문장 안에 `美食` 2회 | 두 번째는 `好地方`, `這裡` 등으로 교체 |

## Step 2 — 언어별 패턴 오류

### 영어

| 오류 유형 | 잘못된 예 | 수정 예 | 이유 |
|---------|---------|--------|-----|
| 장황한 진행형 도입 | `I was searching for restaurants near me and stumbled upon` | `I looked up restaurants near me and stumbled upon` | 검색어를 그대로 입 밖에 내뱉는 로봇 느낌 |
| 장황한 진행형 도입 | `I was looking for the best restaurants near me` | `Trying to find the best restaurants near me,` | 분사구문으로 교체 |
| 장황한 진행형 도입 | `I've tried quite a few restaurants near me` | `I was checking out restaurants near me on my phone,` | 스마트폰 검색 상황 묘사로 맥락에 녹임 |
| 마무리 문장 중복 | 복수 원고 모두 `Definitely coming back.` | 각 원고 마무리 다르게 — `I'd recommend this to anyone visiting the area.` 등 | 같은 표현 반복 시 템플릿 티 남 |
| 문법 오류 | `I'm so glad I found here` | `I'm so glad I landed here` | here는 장소 부사라 find의 목적어로 쓸 수 없음 |
| 구어체 미흡 | `fits perfectly with` | `fits right in with` | 구어체 교체 |

### 일본어

| 오류 유형 | 잘못된 예 | 수정 예 | 이유 |
|---------|---------|--------|-----|
| 키워드 대명사 범주 오류 | `このレストランは本当におすすめです` | `美味しいレストランを探しているなら、ここは本当におすすめです` | 캐주얼 가게를 `このレストラン`으로 지칭하는 범주 오류. 키워드는 검색의 목적어로만 사용 |
| 키워드 대명사 범주 오류 | `楽しめるレストランだと思います` | `楽しめるお店だと思います` | 동일 원고 내 두 번째 등장 시 お店/ここ로 교체 |
| 키워드 누락 | `このレストランは雰囲気がとても良く` (키워드 없음) | `イテウォンで気になるグルメを探していて立ち寄りました。雰囲気がとても良く` | 키워드 아예 없는 경우 검색 맥락으로 문두 삽입 |
| 지명 중복 | 짧은 원고 안에 `イテウォン` 2회 | 두 번째는 `この辺り`로 교체 | 지명 반복 시 키워드 쑤셔 넣기처럼 보임 |
| 재방문 표현 어색 | `また行きたくなるレストランです` | `また来たくなるお店です` | 가게에 다시 가는 것은 行きたい보다 来たい가 자연스러움 + 키워드 중복 제거 |
| 감탄 표현 밋밋 | `とても満足しました` | `これが大当たりでした` | 발견에 대한 자연스러운 감탄 반응으로 교체 |

### 번체 중국어

| 오류 유형 | 잘못된 예 | 수정 예 | 이유 |
|---------|---------|--------|-----|
| 직역체 | `沒有失望` | `沒讓我失望` | 직역 → 실제 구어 표현 |
| 키워드 중복 | `是會讓人一吃就愛上的美食` (앞에 美食 있음) | `一吃就愛上了` | 같은 문장 안에 동일 키워드 2회 |
| 키워드 중복 | `是會讓人想再訪的美食餐廳` | `是會讓人想再訪的好地方` | 美食 중복 제거. 好地方/這裡로 교체 |
| 어색한 단어 조합 | `披薩的口感非常接近紐約風格` | `披薩很有紐約的感覺` | 口感(식감)+風格(스타일) 조합은 어색. 有～的感覺로 자연스럽게 |

## Step 3 — 현지인 관점 자연스러움

각 원고를 해당 언어 원어민이 읽었을 때 위화감이 없는지 검수한다.
"문법적으로 맞는가"가 아니라 "실제 사업자가 이렇게 쓸까?"가 기준.

### 3-1. AI 패턴 검출

- [ ] 대칭 나열, 수식어 남발, 과한 칭찬 ("완벽한", "최고의")
- [ ] 이모지 외 특수기호 — 대시(-), 별표(*), 슬래시(/) 모두 제외
- [ ] 각 원고가 리뷰 내용을 실제로 반영하는가 — 복붙 느낌 나는 일반 답변 금지
- [ ] 의료 매장: 효과 보장/최상급/전후 비교 표현 없는가

### 3-2. 키워드 삽입 품질

키워드가 "있는가"가 아니라 "자연스러운가"를 검수.

- [ ] 사업자가 실제로 쓸 법한 문맥에 녹아있는가
- [ ] "our [지역] [업종]" 같은 광고 카피 패턴이 아닌가
- [ ] 키워드 삽입 위치가 문장 흐름을 끊지 않는가
- [ ] 서론/결론에 이미 키워드가 있으면 본론에서 억지로 또 넣지 않아도 됨

| 언어 | 부자연스러운 삽입 | 자연스러운 삽입 |
|------|-----------------|---------------|
| EN | our Gangnam makeup salon | here in Gangnam |
| EN | at our Cheongdam beauty clinic | when you're in Cheongdam |
| JP | 江南アートメイクならではの | (결론 태그에 이미 있으므로 본론 삽입 불필요) |
| JP | 江南でのリタッチ | リタッチの際はまたいつでも |
| ZH | 我們江南的半永久化妝店 | 下次來江南的時候 |

### 3-3. 본론 문단 구조 (전 매장 공통)

- [ ] 본론이 3문장 이상인데 1문단으로 붙어 있는가 → 최대 2문단으로 분할 권장
- [ ] 본론이 3문단 이상인가 → 2문단 이내로 축약
- [ ] 분할 지점이 의미 단위인가 (감사·반응 / 매장 스토리·재방문 유도 등 자연스러운 경계)

### 3-4. 이모지 매장 스타일 매칭

- [ ] store-info.md 매장 스타일과 이모지 사용 패턴이 일치하는가
  - 친근·캐주얼 매장: 본문에 이모지 적극 사용
  - 정중·차분 매장: 서론·결론 위주, 본문 절제
- [ ] 매장 지정 이모지 외 임의 이모지 남발 없는가
- [ ] 1문장 1이모지 초과 사용 없는가

### 3-5. 톤/격식 수준

업종별 톤 가이드(구글작업 CLAUDE.md)와 대조.

- [ ] 뷰티/F&B에 과도한 격식체 사용하지 않았는가 (光栄です, 幸甚です 등)
- [ ] 의료에 과도한 캐주얼 사용하지 않았는가
- [ ] 리뷰어의 톤과 답변 톤이 극단적으로 불일치하지 않는가 (캐주얼 리뷰에 격식 답변 등)

| 업종 | 피해야 할 표현 | 적절한 대체 |
|------|--------------|-----------|
| 뷰티 | 光栄です、恐縮です | 嬉しいです、よかったです、ありがとうございます |
| 뷰티 | We are honored | We're so glad, Happy to hear |
| F&B | 誠にありがとうございます | ありがとうございます、嬉しいです |
| 의료 | めっちゃ嬉しい | 嬉しく思います |

## Step 4 — 복수 원고 교차 검수

배치 내 원고 전체를 놓고 패턴 반복을 검출한다.
개별 원고가 괜찮아도 묶어서 보면 템플릿 티가 나는 경우를 잡는 단계.

### 4-1. 표현 중복

- [ ] 같은 감정 표현이 3건 이상 반복되는가 (嬉しいです, So glad, 很高興 등)
- [ ] 마무리 문장이 동일 패턴인가 (Definitely coming back / またぜひお越しください)
- [ ] 서론 직후 첫 문장이 같은 구조인가

중복 발견 시: 동의어/다른 문장 구조로 분산

| 원래 반복 | 분산 대체 풀 |
|---------|------------|
| 嬉しいです (x3) | 嬉しいです / よかったです / ありがたいです / 光栄です(의료만) |
| So happy / So glad (x2) | So happy / Really glad / Great to hear / Love that |
| 很高興 (x2) | 很高興 / 很開心 / 太好了 |

### 4-2. 구조 반복

- [ ] 문장 수, 문단 길이가 모든 원고에서 거의 동일하지 않은가
- [ ] 키워드 삽입 위치가 매번 같은 자리(끝에서 두 번째 문장 등)인가

## 출력 형식

```
[검수 결과]

Step 1 키워드 정합성
- #원고번호: (문제 내용) → (수정 방향)

Step 2 언어별 패턴 오류
- #원고번호 (언어): (오류 유형) — 수정 전: "..." → 수정 후: "..."

Step 3 현지인 관점 자연스러움
- 3-1 AI 패턴: #원고번호 (문제 내용)
- 3-2 키워드 삽입 품질: #원고번호 "문제 표현" → "자연스러운 대체"
- 3-3 톤/격식: #원고번호 (업종 대비 부적절한 표현)

Step 4 복수 원고 교차 검수
- 4-1 표현 중복: "표현" x 등장횟수 → 분산 방안
- 4-2 구조 반복: (패턴 설명)

[수정 원고]
문제 있는 원고만 재작성. 문제 없는 원고는 "이상 없음" 표기.
```
