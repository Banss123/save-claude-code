# GBP 통합 보고서 생성 (비즈하이 버전)

## Overview

GMB Everywhere PDF 3개를 인풋으로 받아 하나의 통합 진단 보고서를 생성한다.
보고서 목적: 메타광고로 유입된 병의원/뷰티 잠재 고객에게 미팅 없이 전달하는 세일즈 자료.
진단 자체로 전문성을 증명하고, 클라이언트가 먼저 연락하게 유도하는 구조.

## 입력

GMB Everywhere에서 export한 PDF 3개:
- Basic Audit — 카테고리, 서비스, 운영시간, 리뷰 수/평점, NAP
- Website Audit — NAP 불일치, 타이틀/메타 분석, 카테고리 페이지 연결
- Review Audit — 응답률, 리뷰 유입 속도, 키워드 클라우드

추가 입력 (PDF에 없는 경우 사용자에게 확인):
- 타겟 국가 (없으면 리뷰 언어 분포 기준으로 추론)
- 주력 시술/서비스 (없으면 서비스 목록 + 리뷰 키워드 기준으로 추론)
- 상권 특성 (없으면 주소 기반 추론)

## 작성 원칙

- 수치는 PDF 데이터 그대로 사용 — 추측 금지
- 리뷰 가이드는 실제로 써도 자연스러운 현지 구어체로 작성
- Zone 분석은 주소/업종 기반으로 실제 상권 특성 반영
- 구매 장벽은 리뷰 키워드 클라우드 + 부정 리뷰 내용 기반으로 도출
- 의료 매장: 구글작업 CLAUDE.md 의료법 광고 규제 섹션 준수 (효과 보장, 최상급 표현 금지)
- 비즈하이 서비스 범위 엄수: 리뷰 CS 응대(답글)만 대행. 리뷰 발행·요청 시스템 구축은 비즈하이 서비스 아님.
- 리뷰 답글 시간 약속 금지: "48시간 내" 등 구체적 응답 시간을 명시하지 마라. 작업자 상황에 따라 달라지므로 "정기 답글 운영"으로 표현.
- 리뷰 비중 과다 금지: 보고서 전체가 리뷰 중심이 되지 않도록 GBP 최적화·포스팅 운영 항목과 균형 유지.
- 합쇼체 필수: 보고서 전체를 "~입니다", "~됩니다", "~있습니다" 합쇼체로 작성. "~한다", "~이다", "~진다" 평서문 종결체 금지. 콜드 아웃리치 자료이므로 읽는 사람에게 공손한 톤 유지.
- 실행 로드맵, 서비스 제안 테이블, 포스팅 캘린더는 보고서에 포함하지 마라. 보고서는 진단 + 전략까지만. 실행은 계약 후 별도 제공.
- 랭킹 팩터 테이블에 출처명(Whitespark 등), 순위, 점수를 노출하지 마라. 팩터명 + 현재 상태만 표시.

## 담당자명 처리

사용자가 담당자명을 별도로 지정하지 않으면 Auditor 필드를 "주식회사 비즈하이"로 기입.

## 저장 경로 규칙

보고서 저장 전 `구글SEO/` 하위에 해당 매장 폴더가 존재하는지 확인 (작업자 매장은 루트에 `N. 매장명`, 영업자 매장은 `[영업자]/매장명`).

- 작업자 매장 → `구글SEO/{N. 매장명}/보고서/` 에 저장 (N은 1~11 순번 prefix, 매장명 부분 일치로 폴더 탐색)
- 영업자 매장 → `구글SEO/[영업자]/{매장명}/보고서/` 에 저장
- 폴더 없음(미등록) → `구글SEO/보고서/매장명/` 에 저장 (메타 DB 유입 잠재 고객 보관용)
- 파일명: `[매장명]_GBP통합보고서_[날짜].md`
