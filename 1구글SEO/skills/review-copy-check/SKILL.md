# 리뷰 원고 검수

## Overview

구글맵에 올릴 방문객 리뷰 원고를 받아 AI/인위적 리뷰 감지 위험, 키워드 포함 여부, 구어체 자연스러움, 배치 패턴 중복, 의료법 위반을 검수한다.
리뷰 답변 원고 검수는 `review-check` 스킬 사용.

## 실행 순서

1. 매장명으로 `store-info.md` 읽기 — 키워드, 업종, 다국어 여부 확인
2. 의료 업종(의원/치과)이면 `skills/_shared/의료법규제.md` 읽기
3. Step 1 → Step 2 → Step 3 → Step 4 순서로 전체 원고 검수
4. 문제 항목을 심각도 순으로 정리
5. 수정 원고 제공 (문제 항목만 재작성)

---

## 해석본 요건

모든 외국어 원고(영어 포함)에 한국어 해석본을 원고 바로 아래 첨부한다.
한국어 원고만 해석본 생략 가능.

---

## Step 1 — 키워드 포함 여부

리뷰 원고의 SEO 목적은 키워드를 자연스럽게 심는 것. 빠지면 의미 없음.

### 체크 항목

- [ ] store-info.md 지정 키워드(업종/지역/서비스)가 배치 전체에 분산 삽입되어 있는가
- [ ] 키워드가 한 원고에 2회 이상 중복 등장하는가 → 두 번째부터 유의어/지시어로 교체
- [ ] 키워드 삽입 위치가 매번 동일한 자리인가 (끝에서 두 번째 문장 등) → 분산 필요
- [ ] 키워드가 문맥에 자연스럽게 녹아있는가, 아니면 쑤셔 넣은 느낌인가

---

## Step 2 — 매장명 직접 언급 (원칙: 금지)

리뷰 원고는 SEO 목적으로 작성되는 가짜 방문객 리뷰. 매장명이 본문에 들어가면 즉시 인위적 작성 티가 드러나고 광고/대행 의심을 받는다. 실제 오가닉 방문객 중 매장명을 본문에 직접 언급하는 비율은 매우 낮음.

### 원칙

- **배치 내 매장명 직접 언급은 0건이 기본**. "Bellarsi", "벨라르시 광화문점", "海木景福宮店" 같은 고유명사 사용 금지
- 매장 지칭이 필요하면 지시어/일반 명사로 대체:
  - 한국어: "여기", "이 미용실", "이 가게"
  - 영어: "this place", "here", "the salon"
  - 일본어: "ここ", "お店", "この店"
  - 번체: "這裡", "這家店"
- 발견 즉시 수정. 한 건도 통과시키지 마라

### 체크 항목

- [ ] 배치 내 매장명 직접 언급이 0건인가
- [ ] 한국어/외국어 해석본 양쪽 모두에서 제거됐는가 (해석본에만 남는 케이스 잦음)

### 실패 사례

- 2026-05-07 벨라르시 후기원고 10건 작성 시 영어 후기 3건에 "Bellarsi", "Bellarsi Gwanghwamun" 노출. 사용자 지적으로 전건 재작성. 이 게이트 강화 이유.

---

## Step 3 — 배치 패턴 중복 검출

개별 원고가 괜찮아도 묶어서 보면 템플릿 티가 나는 경우를 잡는다.

### 3-1. 마무리 문장 패턴

- [ ] 마무리 문장이 동일 구조인가 (키워드 + 추천 CTA 반복 등)
- [ ] 추천 표현이 3건 이상 동일한가 ("강추합니다", "おすすめです" 등)

마무리 패턴이 동일하면 배치 전체가 동시에 올라갈 때 즉시 스팸 감지 대상.

### 3-2. 내용 흐름 패턴

- [ ] 배치 내 3건 이상 동일한 전개 구조인가 (상담 꼼꼼 → 결과 자연스러움 등)
- [ ] 시술 전 불안 → 친절한 안내 → 만족 구조가 반복되는가

각 원고마다 다른 방문 동기 / 경험 포인트 / 감정 흐름으로 변주할 것.

---

## Step 4 — AI 패턴 및 구어체 자연스러움

"문법적으로 맞는가"가 아니라 "실제 방문객이 이렇게 쓸까?"가 기준.

### 공통 구어체 체크

- [ ] 실제 방문객이 쓸 법한 구어체로 작성됐는가 — 문어체, 공문서체, 기업 홍보 문체 사용 여부 전반 확인
- [ ] 개인 경험 서술인가, 매장을 외부에서 평가하는 문장인가

### 공통 AI 패턴

- [ ] 광고성 평가 문체 — 매장을 외부에서 객관 평가하는 문장 ("技術力が高い", "기술력이 뛰어난") → 개인 경험 표현으로 전환
- [ ] 과한 칭찬/과장 표현 ("완벽한", "최고의", "믿을 수 없는")
- [ ] 대칭 나열, 수식어 남발 구조
- [ ] CTA성 마무리 과다 ("꼭 가보세요", "강추합니다") — 배치 내 반복 금지

### 언어 혼용 감지 (전 언어 공통)

- [ ] 원고 작성 언어와 다른 문자 체계가 혼용되어 있는가 — 예: 일본어 원고에 한글, 영어 원고에 한글
- [ ] 발견 즉시 수정. 메뉴명은 해당 언어 표기 규칙으로 전환:
  - 일본어: 한글 메뉴명 → 가타카나 음독 또는 일본어 의역. 예: 갈비살 → カルビ / 양념갈비살 → ヤンニョムカルビ / 꽃목살 → コッモクサル / 한돈모듬스페셜 → ハンドン盛り合わせ
  - 영어: 한글 메뉴명 → 로마자 음독. 예: 갈비살 → galbisal / 양념갈비살 → yangnyeom galbisal (로마자는 허용, 한글 자체 삽입은 금지)
  - 번체: 한글 메뉴명 → 한자 의역 또는 주음 표기

### 한국어

- [ ] `-요체`와 `-습니다체` 혼용 여부 → 하나로 통일
- [ ] 마지막 문장이 광고 카피처럼 읽히는가 (키워드 + 추천 구조)

### 영어

- [ ] 대시 계열 기호 사용 여부 — em dash(—), en dash(–), double dash(--) 전부 금지. 즉시 마침표 또는 쉼표+접속사로 교체. 실제 방문객 리뷰에서도 대시는 AI 작성 티가 강하게 드러남
- [ ] "exactly what I was hoping for" / "couldn't be happier" / "highly recommend" 클리셰 반복 여부 → 배치 내 2회 이상 금지
- [ ] 첫 문장이 "I finally found..." / "If you're looking for..." 도입 패턴 반복 여부 → 배치 내 분산

### 일본어

- [ ] 문어체 표현 ("施術を受けることができました" → "受けてきました")
- [ ] 광고성 평가 문체 ("デザインセンスがとても良く" 등) → 개인 경험 표현으로 전환
- [ ] 키워드를 가게 지칭 대명사로 쓰는 범주 오류 ("このレストラン" → "お店", "ここ")
- [ ] 재방문 표현: "また行きたい" → "また来たい"

### 번체 중국어

- [ ] 직역체 ("沒有失望" → "沒讓我失望")
- [ ] 키워드 중복 ("美食" 2회 등) → "好地方", "這裡"로 교체

---

## Step 5 — 의료법 체크 (의원/치과 업종만)

`skills/_shared/의료법규제.md` 기준 적용.

- [ ] 시술 효과 보장 표현 없는가 ("확실히 나아졌어요", "완치됐어요")
- [ ] 최상급 표현 없는가 ("최고의 병원", "가장 잘하는")
- [ ] 전후 비교 표현 없는가 ("시술 전엔 OO였는데 지금은 XX")
- [ ] 타 병원 비교/비방 없는가

위반 발견 시 즉시 경고하고 대체 표현 제시. 원본 그대로 통과시키지 마라.

## 출력 형식

```
[검수 결과]

Step 1 키워드 포함
- 삽입 현황: 키워드별 등장 분포 요약
- 문제: #번호 (누락/중복/위치 고정/부자연스러운 삽입)

Step 2 매장명 직접 언급
- 언급 건수: X건 (원칙: 0건)
- 문제 원고: #번호 → 대체 지시어 제안 ("여기", "this place" 등)

Step 3 배치 패턴 중복
- 3-1 마무리 패턴: "표현" x 등장횟수 → 분산 방안
- 3-2 내용 흐름: 반복 구조 설명

Step 4 AI 패턴 및 구어체
- 언어 혼용: #번호 혼용 문자 → 수정 표기
- 공통: #번호 (문제 내용) → 수정 방향
- 한국어: #번호 (문제 내용) → 수정 방향
- 일본어: #번호 수정 전: "..." → 수정 후: "..."
- 번체: #번호 (문제 내용) → 수정 방향

Step 5 의료법 (해당 업종만)
- #번호: 위반 표현 → 대체 표현

[수정 원고]
문제 있는 원고만 재작성. 문제 없는 원고는 "이상 없음" 표기.
```
