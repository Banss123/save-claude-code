# 리뷰 답변 생성

## Overview

리뷰 원문을 입력받아 매장별 포맷에 맞는 답변 원고를 생성한다.
복수 리뷰를 한번에 처리. 리뷰 원문 언어 기준으로 답변 작성.

## 🚫 MANDATORY PRE-OUTPUT GATE (누락 시 출력 무효)

아래 조건 중 하나라도 미충족이면 출력하지 말고 다시 작성:

1. **외국어 답변 → 한국어 해석본 첨부**: 한국어가 아닌 모든 답변(영어/일본어/중국어/기타) 바로 아래에 `--- 한국어 해석 ---` 또는 `[한국어 해석]` 블록으로 한국어 번역 첨부. 영어 전용 매장도 예외 없음. 업주는 한국인이므로 답변 내용 확인용 필수.
2. **파일 저장 시에도 해석본 포함**: 콘솔 출력에만 붙이고 파일 저장 시 누락하지 않도록. 파일 내용과 출력 내용 동일해야 함.
3. **⚠️ 외국어 답변 본문에 한국어 침투 절대 금지**: 게시될 답변 본문에는 해당 외국어 외 그 어떤 한국어 문자도 들어가서는 안 됨. 키워드 옆 괄호 한국어 병기(예: `ホンデ 皮膚科(홍대 피부과)`, `english speaking dermatologist(영어 가능 피부과)`) 본문 삽입 절대 금지. 한국어 병기는 **하단 [사용 키워드] 블록 또는 [한국어 해석] 블록에서만** 허용. 본문은 100% 해당 외국어 + 숫자 + 이모지 + 라틴 알파벳·매장 영문명만 허용 (단 매장명에 한국어 포함 시 제외).
   - 검사 방법: 본문에서 정규식 `[가-힣]` 1자라도 매칭되면 즉시 재작성
   - 실패 사례: 2026-05-11 셀린의원 홍대점 일본어 리뷰 답변에 `ホンデ 皮膚科(홍대 피부과)` 그대로 게시용 본문에 삽입 → 일본 고객이 답변 보면 한국어 괄호 노출 → 재작성 발생. 이 게이트 3번 추가 이유

실패 사례: 2026-04-22 에버피부과의원 리뷰 20건 작업 시 영어 답변 7건 해석본 누락 → 재작업 발생. 이 게이트 추가 이유.

## 실행 순서

1. 매장명으로 해당 `store-info.md` 읽기
2. `리뷰 답변 포맷` (서론/결론/톤/줄바꿈 규칙) 확인
3. 각 리뷰의 언어 감지
4. 언어별 답변 생성. 리뷰 원문 언어와 동일하게
   - ⚠️ store-info.md의 `작업 언어` 목록은 소식글·썸네일 작성 기준임. 리뷰 답변 언어를 제한하지 않음
   - 터키어·프랑스어·베트남어 등 작업 언어 목록에 없는 언어로 쓰인 리뷰도 해당 언어로 답변. 영어 등 다른 언어로 대체 금지
   - 실패 사례: 2026-05-03 셀린의원 명동점 터키어 리뷰 2건을 "작업 언어 아님"으로 판단해 영어 답변 → 재작업 발생
5. ⚠️ **[MANDATORY]** 모든 비한국어 답변 바로 아래에 `--- 한국어 해석 ---` 블록으로 한국어 해석본 첨부. 영어 주력 매장도 예외 없음 (업주 보고용). 이 단계 누락 시 출력 무효 → 다시 작성
6. 리뷰 번호 또는 순서 표기해서 묶어서 출력
7. 원고 하단에 사용된 키워드 목록 명시 (업종/지역/서비스 키워드 구분)
8. **외국어 키워드 괄호 한국어 병기는 하단 [사용 키워드] 블록 또는 [한국어 해석] 블록 내에서만**. 게시용 답변 본문에는 절대 금지. 예: `ホンデ 皮膚科(홍대 피부과)` 표기는 키워드 목록·해석본에서만 OK, 일본어 본문에는 `ホンデ 皮膚科`만 들어가야 함
9. 출력 전 빠른 4점 체크 (실패 빈도 높은 항목만. 나머지는 review-check에 위임):
   - 비한국어 답변 전체에 `--- 한국어 해석 ---` 블록 첨부됐는가 (영어 포함)
   - **⚠️ 외국어 본문에 한국어(가-힣) 침투 없는가** — 키워드 옆 괄호 한국어 병기, 한국어 단어, 한국어 조사 등 단 1자라도 본문에 있으면 즉시 제거. [한국어 해석] 블록 내부만 예외
   - 대시 계열(—, –, --) 하나라도 있으면 즉시 수정
   - 키워드가 이 매장·지점 지정 키워드인가 (타 지점 혼용 주의)
10. `review-check` 스킬 자동 실행. 생성 완료 즉시 검수 진행. 문제 없으면 "검수 이상 없음" 한 줄로 표기. 문제 있으면 수정본으로 교체 후 최종 출력.

⚠️ **출력·저장 전 한국어 침투 정규식 자가 점검 (필수)**: 외국어 답변 본문 영역(서론·본론·결론, [한국어 해석] 블록과 [사용 키워드] 블록 제외)을 한 줄씩 훑어 `[가-힣]` 패턴이 매칭되는지 확인. 1자라도 있으면 출력 무효 → 본문에서 한국어 제거 후 재작성. 검수 통과 보고 전에 반드시 수행.

## 원고 작성 원칙

### 톤
- 구글맵, 샤오홍슈, 코네스트 등 실제 방문객의 오가닉 리뷰처럼 자연스럽게
- 과한 칭찬 X. "너무너무 감사합니다", "정말 행복합니다" 류 금지
- 과장 표현 X. "최고의", "완벽한", "믿을 수 없는" 류 금지
- AI가 쓴 것처럼 느껴지는 문장 구조 금지 (대칭 나열, 수식어 남발)
- 이모지를 제외한 특수기호 사용 금지 (별표, 슬래시 등 모두 제외)
- 대시 사용 절대 금지: em dash(—), en dash(–), double dash(--), 구분선(---) 전부 포함. 영어/일본어/중국어 원고도 동일 적용. 문장 연결이나 부연 설명에 대시 쓰면 AI 작성 티가 즉시 드러남. 반드시 마침표, 쉼표, 접속사로 대체할 것
- 자연스러운 구어체. 문어체 X
- ⚠️ **업주 시점 유지 필수**: 답변 전체는 매장(업주) 관점으로 일관되게 작성. 고객 체험 감상 표현 금지. 아래 유형은 즉시 재작성:
  - "~줄 몰랐다", "~놀랐다", "~새로웠다" — 업주가 자신의 서비스에 대해 처음 발견하거나 놀라는 표현
  - "체험해보고 ~", "실際に体験して ~" — 업주가 고객처럼 자신의 서비스를 체험하는 관점
  - 실패 사례: "AI肌診断でここまで詳しく肌の状態がわかるとは思っていなかったので、実際に体験してとても新鮮でした" → 업주가 자신의 AI 진단에 놀라는 고객 시점 → 재작성

### AI 탄로나는 금지 표현 (반복 시 즉시 AI 작성 티)

**영어 본론 구조**
- 본론은 단일 문단으로 작성. 두 문단으로 나누지 않음
- 첫 문장은 방문 감사 표현으로 자연스럽게 시작. "We're glad to hear...", "So happy to hear...", "Glad to hear...", "We're pleased to hear..." 류 AI 패턴 금지
- 감사 표현 이후 리뷰 포인트 반응 + 매장 스토리를 하나의 흐름으로 이어 붙이기. 설명 위주 첫 문단 + 반응 위주 두 번째 문단으로 나누지 말 것

**영어 자기 과장 표현 금지**
- "exactly what we aim for" / "exactly what we're here for" / "That's what we do best" / "We work hard to..." / "We put a lot of care into..."
- 매장이 자기 자신을 설명·홍보하는 정형 마케팅 톤

**고객 칭찬 그대로 되받아치기 금지**
- "Being called the best [리뷰 원문 칭찬] is a real compliment" 식
- 고객의 칭찬 문구를 그대로 인용해서 자화자찬으로 되받아치는 건 AI 정형 패턴

**한국어 첫 문장 방향**
- 방문에 대한 감사/기쁨을 자연스럽게 담는 것 OK. "방문해 주셔서 감사합니다" 계열 표현은 허용하되, 리뷰 내용에 맞게 구체화할 것
- 금지: 감사 1문장만 달랑 쓰고 끝내거나, 리뷰와 무관하게 붙여넣은 듯한 범용 감사 문구
- 매장 운영 디테일은 감사 맥락을 보강할 때만 사용. 본론 전체를 매장 설명으로 채우지 말 것

### 구조
- 서론 / 본론 / 결론 사이 한 줄 띄움
- 매장별 지정 서론/결론 포맷 100% 준수
- ⚠️ store-info.md 서론 항목이 `(없음 — ○○으로 바로 시작)` 형식인 경우: 괄호 안 지시를 반드시 따를 것. "없음"을 "아무것도 쓰지 않는다"로 해석 금지. 예: `없음 — 감사 표현으로 바로 시작` = 첫 문장은 감사 표현이어야 함
- 실패 사례: 2026-05-03 셀린의원 홍대점 일본어 답변, 서론 `없음`을 인삿말 전체 생략으로 해석 → 본론 직접 시작 → 재작성 발생

### 본론 문단 분할 (전 매장 공통)
- **본론 2문장 이하 → 1문단**, **3문장 이상 → 최대 2문단으로 분할**. 3문단 이상 절대 금지
- 분할 기준: 의미 단위로 자연스럽게 나눔 (감사·반응 / 매장 스토리·재방문 유도 등). 억지 분할 금지
- 모바일에서 한 덩어리로 보이는 4~5문장 벽은 가독성 저하. 의료/뷰티 매장도 동일 적용
- 문단 간 한 줄 띄움

### 이모지 사용 (매장 스타일 매칭 필수)
- store-info.md `리뷰 답변 포맷` 항목의 매장 톤·이모지 사용 패턴을 그대로 따른다
  - **친근·캐주얼 매장** (예: 셀린의원 명동점, F&B): 서론·본론·결론 전반에 이모지 적극 사용 (💙 ❤️ ✨ 😊 💖). 문장 끝 이모지 권장
  - **정중·차분 매장** (예: 셀린의원 홍대점): 서론·결론에만 매장 지정 이모지 1~2개. 본론은 이모지 없거나 극히 절제
- 매장 스타일이 store-info.md에 명시 안 된 경우 기존 작업 이력(`리뷰답변/` 폴더 최신 파일) 패턴 따름
- 과한 이모지 남발 금지 (1문장 1이모지 초과)

### 본론 작성 실전 원칙

**분량**
- 본론 2~4문장 필수. 1문장으로 끝내지 말 것. 리뷰에 성의 있게 반응하려면 구체성이 필요
- 너무 길게 늘어뜨리지도 말 것 (5문장 이상이면 과함)

**첫 문장**
- 본론 비중은 감사/방문 반응이 주(6), 메뉴/경험 설명이 보조(4). 비율로 보면 6:4 정도
- 감사 표현은 리뷰 내용에 맞게 구체화할 것. "방문해 주신 것 정말 기뻤습니다" 같은 방문 자체에 대한 감사 허용
- 매장 설명이 필요하면 감사 맥락을 보강하는 역할로만 1문장 이내
- 복수 리뷰 간 시작 문장 크로스체크. 동일 패턴 반복 있으면 재작성

**매장 고유 스토리 자연 삽입**
- 운영 철학, 작업 방식, 메뉴 특징, 공간 구조 등을 구체적으로 언급
- 예시: "매일 아침 오픈 전에 손질해서 나가고", "8 테이블 구조라 웨이팅이 길어지니 조금 일찍 오시면", "수란 노른자가 고기를 감싸는 느낌을 살리려고 일부러 소스와 맞춰"

### 리뷰 포인트 반응 방식

- 리뷰의 구체 포인트(서비스, 메뉴, 분위기, 직원 태도 등) 중 1~2개 선택해서 언급
- 리뷰 칭찬을 그대로 인용해서 되받아치기 금지. 매장 관점에서 "왜 그런 결과가 나왔는지" 자연스럽게 설명
- 예시:
  - 리뷰: "정말 부드러웠다"
  - 나쁨: "Being called 'very tender' means a lot"
  - 좋음: "토시살과 갈비살은 매일 아침 손질해서 나가고, 테이블에서 알맞은 불 세기로 구워드리기 때문에 육질이 부드럽게 유지됩니다"

### 한국어 톤 기본값 (매장 미지정 시)

매장 store-info.md에 한국어 답변 톤이 별도 지정되지 않은 경우 기본 적용:

- 정중 업주 답글체
- 필수 표현 풀:
  - "~ 진심으로 감사드립니다"
  - "~ 저희에게 큰 힘이 됩니다"
  - "그 진심이 잘 가 닿은 것 같아 큰 보람을 느끼며"
  - "변함없는 [매장 특징]으로 모실 수 있도록 준비하고 기다리겠습니다"
- 매장 운영 철학/메뉴 스토리를 본론에 자연스럽게 녹이기

### 키워드
- store-info.md의 업종/지역 키워드 전체 반드시 포함. 일부만 넣는 것 금지
- 지역명 + 핵심 서비스명 자연스럽게 1~2회 포함
- 리뷰 원고에 매장명 삽입 금지. 실제 방문객 리뷰처럼 보이지 않음
- 키워드 밀도 과다 금지 (스팸 감지 위험)
- 같은 리뷰 안에 동일 키워드 반복 금지. 두 번째 등장 시 유의어/지시어로 교체

### 키워드 삽입 방식 (언어별)

**영어**
- `I was searching for [keyword]`, `I was looking for [keyword]` 같은 장황한 진행형 도입 금지. 검색어를 그대로 입 밖에 내뱉는 로봇 느낌
- 대신: `I looked up [keyword]`, `While looking up [keyword]`, 분사구문, 스마트폰 검색 상황 묘사(`I was checking out [keyword] on my phone`) 등으로 자연스럽게 맥락에 녹이기
- 복수 리뷰에서 마무리 문장 중복 금지. `Definitely coming back` 등 동일 표현이 반복되면 템플릿 티가 남

**일본어**
- 캐주얼한 식당(피자집, 고깃집 등)을 `このレストラン`으로 지칭하는 것은 범주 오류. `お店`, `ここ`, `ピザ屋` 등 사용
- `レストラン`, `グルメ` 등 키워드는 가게를 지칭하는 대명사가 아닌 검색의 목적어로만 사용. "○○を探していて見つけました" 구조
- 같은 리뷰 안에 키워드 2회 등장 시 두 번째는 `お店` 또는 `ここ`로 교체

**번체**
- 키워드(`美食` 등)가 같은 문장 안에 2회 등장하지 않도록. `好地方`, `這裡` 등으로 교체
- `沒有失望` 같은 직역체 금지. `沒讓我失望` 등 실제 구어 표현 사용
- `口感` + `風格` 조합처럼 어울리지 않는 단어 결합 주의. `很有～的感覺` 등 자연스러운 표현으로

## 🛡️ 1점 방어 리뷰 대응 플로우

1점(부정) 리뷰는 별도 플로우. **구체 해명·반박 회피 + commitment 톤** 원칙.

### 6단계 구조

1. **매장 기본 서론** (store-info.md 포맷 100% 일치)
2. **사과 + 리뷰어 지적 포인트 언급** (해명 없이 수용하는 톤)
   - 예: "We're sincerely sorry that your experience did not meet your expectations, especially regarding [리뷰어 언급 포인트]."
3. **"We will" 구조로 commitment 표현** (현재형 "we take" 대신 미래형 "we will take")
   - 예: "We will take feedback like yours seriously and look into it more closely."
4. **직접 연락 유도 문장**
   - 예: "Please kindly reach out to us directly so we can better understand what happened."
5. **"We will" + together/work toward making things right 마무리**
   - 예: "and we will work with you toward making things right."
6. **매장 기본 결론** (store-info.md 포맷 100% 일치)

### 금지 사항

- "exactly what we aim for" / "We work hard to..." 류 자기 과장 표현
- "We appreciate all feedback" / "Your feedback is important to us" 류 제네릭 감사 (commitment 약함)
- 구체 상황 반박 ("The staff was actually friendly but...") — 공개 공방 유발
- 할인·보상 공개 약속 (별도 채널로 조율)
- 리뷰어 표현 그대로 되받아치기 ("Being called unprofessional...")

### 톤 기본 원칙

- **진심 사과 + 행동 commitment** 균형
- 고객 → 매장 비공개 채널 이동 유도
- 다른 잠재 고객 읽는 것도 의식 (공정한 매장으로 보이도록)

### 4-Block 출력 구조 (⚠️ 1점 방어 리뷰 전용 — 일반 긍정 리뷰 적용 금지)

업주(한국인) 확인 편의를 위해 다음 4블록 순서로 출력:

```
--- 리뷰 원본 ([언어]) ---
(리뷰 원문 그대로)

--- 리뷰 원본 한국어 해석 ---
(리뷰 원문 한국어 번역)

--- 답변 ([언어], Google 게시용) ---
(게시용 답변)

--- 답변 한국어 해석 ---
(답변 한국어 번역)
```

한국어 리뷰는 원본 그대로 + 답변만 작성 (2-block).

- 에버피부과의원 압구정: Template A (최소 deflection, store-info.md 참조) / Template B (상세 해명형, 조건부 사용)

## 출력 형식

⚠️ **출력 전 필수**: 모든 비한국어 답변(영어 포함)에 `--- 한국어 해석 ---` 블록 첨부. 누락 시 출력 무효.

헤더: `[리뷰 N] 언어명 — 리뷰어이름` (예: `[리뷰 1] English — Ra Pony`, `[리뷰 2] 日本語 — Ayumi W.`, `[리뷰 3] 한국어 — 홍길동`)
외국어 답변은 결론 바로 아래 `[한국어 해석]` 블록으로 서론+본론+결론 전체 번역 첨부.
4-block 구조(`--- 리뷰 원본 ---` / `--- 답변 ---` 형식)는 1점 방어 리뷰 전용. 일반 긍정 리뷰에서 사용 금지.

```
[리뷰 1] English — Ra Pony

(본론)

(결론)

[한국어 해석]
(서론)

(본론)

(결론)

---

[리뷰 2] 한국어 — 홍길동

(서론)

(본론)

(결론)

---
```

## 파일 저장 (필수)

리뷰 답변 완성 후, 해당 매장의 리뷰답변 폴더에 텍스트 파일로 저장한다.

- 경로: `구글SEO/[영업자 or 작업자] 매장명/리뷰답변/`
- 파일명: `YYYY-MM-DD 리뷰답변.txt` (작업일 기준)
- 같은 날짜에 이미 파일이 있으면 기존 파일 하단에 구분선(---) 후 추가
- 파일 내 리뷰 간 구분: `---`
- 파일 하단에 사용 키워드 목록 포함
