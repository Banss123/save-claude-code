# 신규 매장 온보딩

## Overview

신규 매장 정보를 입력받아 전체 온보딩 패키지를 한번에 생성하고 `store-info.md`를 만든다.
개별 항목 따로 요청 기다리지 마라 — 전체 묶어서 한번에 제공.

## 입력 수집

아래 정보가 없으면 먼저 물어봐라:

- 매장명
- 업종
- 지역 (역명 또는 동네)
- 계약 유형 (작업자 / 영업자)
- 작업 언어 (예: 한국어, 영어, 일본어)
- 주력 서비스 또는 메뉴
- 의료법 적용 여부 (의원/치과 여부)

## 실행 순서

1. 입력 정보 확인
2. 온보딩 패키지 전체 생성 (아래 5개 항목)
3. 매장 폴더 생성 (계약 유형별):
   - 작업자 매장: `구글SEO/{N. 매장명}/` (N은 현재 사용 중 마지막 번호 + 1, 현재 11까지 사용 중이므로 신규는 12부터)
   - 영업자 매장: `구글SEO/[영업자]/{매장명}/` (번호 prefix 없음)
4. `store-info.md` 생성

## 폴더 구조

작업자 매장 (루트 위치, 숫자 prefix):
```
N. 매장명/
├── store-info.md
└── 리뷰답변/
```

작업자 매장 중 주간보고 대상 (재주좋은치과, 벨라르시, 셀린의원 홍대점, 셀린의원 명동점):
```
N. 매장명/
├── store-info.md
├── 리뷰답변/
└── 주간보고/
```

영업자 매장 ([영업자]/ 하위, prefix 없음):
```
[영업자]/매장명/
├── store-info.md
└── 컨설팅/
```

## 온보딩 패키지 (5개 항목 한번에 출력)

### 1. 키워드 추천 + 전략 컨설팅

**선정 기준**
- 검색량 vs 경쟁도 균형: 검색량 높아도 경쟁 포화 키워드는 후순위. 중간 경쟁도 + 명확한 검색 의도 키워드 우선
- 롱테일 우선: 단일 키워드("치과")보다 복합("강남역 치과 임플란트")이 전환율 높고 순위 잡기 쉬움
- 구글맵 로컬 SEO 특성: 지역명 포함 필수. 역명/동네명 둘 다 커버
- 외국인 타겟 매장: 현지인이 실제 검색하는 표현 기준 (번역 키워드 X, 현지 검색어 O)

**업종별 키워드 전략**
- F&B: 지역+맛집, 메뉴명, 분위기 키워드 (데이트, 혼밥 등) 조합
- 의료/뷰티: 시술명 + 지역, 증상/고민 키워드 (예: "강남 점빼기", "홍대 리프팅")
- 헤어/뷰티살롱: 시술명 + 지역, 스타일 키워드
- 숙박/관광 인접: 랜드마크 기준 거리 키워드 (예: "경복궁 근처 맛집")

**출력 형식**
- 업종 키워드 3개 / 지역 키워드 3개 / 복합 키워드 3개
- 각 키워드마다 용도 명시 (프로필 설명 / 소식글 / 리뷰 답변)
- 우선순위 1~3위 표기 + 이유 한 줄

### 2. 소식글 서론/결론 포맷
- 작업 언어 전체 커버
- 언어별로 서론 / 결론 분리 출력

### 3. 리뷰 답변 서론/결론 포맷
- 작업 언어 전체 커버
- 줄바꿈 규칙 명시 (서론/본론/결론 사이 한 줄 띄움, 본론 내 붙여 쓰기)

### 4. 구글 비즈니스 프로필 설명문 초안
- 300자 이내
- 메인 키워드 자연 삽입
- CTA 포함

### 5. store-info.md 생성
기존 매장 파일 구조 그대로 사용:

```markdown
# [매장명] — 매장 정보

## 기본 정보
- 매장명:
- 업종:
- 지역:
- 계약 유형:
- 다국어 운영:
- 작업 언어:

## 주의사항
-

## 키워드
- 업종 키워드:
- 지역 키워드:
- 외국어 키워드:

## 주력 서비스
(내용)

## 소식글 포맷
(언어별 서론/결론)

## 리뷰 답변 포맷
(언어별 서론/결론/줄바꿈 규칙)

## 작업 이력
(작업 전)
```

## 작성 규칙

- 대시(—, –, --)를 문장 연결이나 부연 설명에 사용하지 마라. 마침표, 쉼표, 접속사로 대체.

## 의료 매장 주의

의료법 적용 대상이면 store-info.md 주의사항에 명시 + 온보딩 패키지 전반에서 의료법 광고 규제 준수.
구글작업 CLAUDE.md 의료법 섹션 참고.
