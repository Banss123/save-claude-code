# CLAUDE.md — 구글작업 프로젝트 규칙

## 역할

응답 스타일 제약: 별표 금지, 대시(—) 문장 연결 금지, 출처 표기 금지. 세부: `skills/_shared/출력-포맷.md`.

## 저장 경로

### 소식글
- 작업자 매장 → `[작업자] 매장명/소식글/YYYYMMDD_주제.txt`
- 영업자 매장 → `[영업자] 매장명/소식글/YYYYMMDD_주제.txt`
- 파일명: 날짜 8자리 + `_` + 핵심 시술/서비스명 (예: `20260410_레이어드컷_C컬매직펌.txt`)
- 저장 형식: 언어 구분 헤더 없이 본문만. 언어 간 구분은 빈 줄 2개

### 보고서
- 등록 매장(폴더 있음) → `[작업자 or 영업자] 매장명/보고서/`
- 미등록 매장(폴더 없음) → `보고서/매장명/` (메타 DB 유입 잠재 고객 보관)

## 매장 구분 (요약)

- **영업자 매장**: 컨설팅·검수 위주, 직접 작업 없음
- **작업자 매장**: 소식글 발행·썸네일·리뷰 답변 등 전 업무 직접 관리

세부 테이블·업종·주의사항: `skills/_shared/매장-목록.md`

## 소식글 직접 작성 프로세스 (작업자 매장)

1. 소재 확인 + 주제 선정 (제안 후 승인)
   - 소재가 주어진 경우 → 바로 2단계 진행
   - 소재가 없는 경우 → 아래 순서로 분석 후 주제 2~3개 추천:
     0) 소식글 폴더 파일 목록 먼저 확인 (store-info.md 이력보다 우선) — 파일이 있으면 store-info.md에 누락된 항목 업데이트 후 분석
     1) 작업 이력 — 최근 발행 주제 패턴 파악, 중복 방지
     2) 메뉴/서비스 목록 — 미발행 항목 추출
     3) 시즌/맥락 — 현재 월·계절·이슈 고려
   - 추천 형식: 주제명 + 추천 이유(시즌·미발행·패턴 환기 중 해당) 1~2줄
   - 작업 이력 또는 메뉴/서비스 목록이 store-info.md에 없으면 → 분석 중단, 사용자에게 해당 정보 요청 후 진행
2. 소식글 원고 작성 (한국어 + 작업 언어)
3. post-check 검수 → 수정 시 재검수 강제 (생략 금지)
4. 파일 저장 (`소식글/YYYYMMDD_주제.txt`)
5. thumbnail-copy 썸네일 문구 생성
6. thumbnail-figma 피그마 자동 적용

세부 규칙은 각 스킬의 SKILL.md 참조.

## 4-way 리뷰 페르소나 (프로젝트 오버라이드)

Step 4(계획 수립)에서 복잡한 작업은 아래 4명으로 압박 테스트. 라우팅은 `~/.claude/.claude/rules/workflow-planning.md`.

| 페르소나 | 시각 | 핵심 질문 |
|---|---|---|
| **SEO전문가** | 검색 노출, 랭킹 팩터 | 이게 GMB 랭킹에 실제로 도움 되나? 의료법/저작권 위반은? |
| **콘텐츠편집자** | 톤, 가독성, 문체 | 별표·대시 안 썼나? 모바일에서 잘 읽히나? 의료법 표현 안 들어갔나? |
| **검색자(사용자)** | 의도, 클릭 동기 | 이걸 검색해서 들어온 사람이 원하는 답이 첫 줄에 있나? |
| **브랜드매니저** | 매장 정체성, 일관성 | 이전 톤과 일관되나? 매장 톤(F&B/의료/뷰티) 맞나? |

라우팅 예시:
- 단순 검수 → 콘텐츠편집자 1명
- 신규 소식글 작성 → 콘텐츠편집자 + 검색자 2명
- 신규 매장 온보딩 → 4명 다

## 에러 처리 (프로젝트 특화)

- **검수 불통과 시**: 수정 지적 3개 이상이면 원고 재작성. 3개 미만이면 피드백만 반환.
- **의료법 위반 표현**: 즉시 경고 + 대체 표현 제시. 원본 그대로 통과 금지. 상세: `_shared/의료법규제.md`.
- **매장 정보 부족 시** (온보딩·프로필 최적화): 필수 정보(업종·주소·메인 서비스) 없으면 작업 중단, 정보 요청. 추측 금지.
- **자동화 제안 거절 시**: 해당 조건 기록, 이후 같은 패턴에서 자동 제안하지 않음.
- **다국어 번역 품질 판단 불가 시**: 원문·번역본 함께 제시, 사용자 검수 요청.

전역 에러 처리 원칙: `~/.claude/.claude/rules/error-recovery.md`
