# Korean Copy Patterns

Use these patterns for Korean headlines, CTAs, landing page sections, social hooks, and product copy. Adapt the pattern to the brand voice. Do not fill unknown proof with fabricated numbers.

## Headline Patterns

### Outcome Without Pain

`[원하는 결과]를 [기존 고통] 없이`

Examples:

- 매주 보고서를 야근 없이 끝냅니다
- 개발자 도움 없이 첫 랜딩페이지를 배포합니다
- 광고비를 늘리지 않고 전환율부터 고칩니다

### Audience + Job

`[타깃]을 위한 [작업/카테고리]`

Examples:

- B2B 마케터를 위한 검색 수요 발굴 워크플로우
- 프랜차이즈 점주를 위한 리뷰 관리 자동화
- 1인 창업자를 위한 콘텐츠 운영 시스템

### Problem First

`[고통스러운 상황], 이제 [새 방식]으로 해결하세요`

Examples:

- 콘텐츠 아이디어가 매주 막힌다면, 고객 질문에서 먼저 찾으세요
- 전환율이 떨어졌다면, 문장보다 약속부터 점검하세요

### Sharp Editorial Claim

`[통념]보다 [더 중요한 기준]이 먼저입니다`

Examples:

- 글감보다 먼저 정해야 할 것은 독자의 상황입니다
- SEO보다 먼저 고쳐야 할 것은 신뢰입니다

## CTA Patterns

Weak CTAs:

- 자세히 보기
- 알아보기
- 문의하기
- 시작하기

Make CTAs specific:

| Intent | Better CTA |
|---|---|
| Trial | 무료로 첫 프로젝트 만들기 |
| Demo | 실제 화면으로 확인하기 |
| Lead magnet | 체크리스트 받기 |
| Consultation | 내 상황 상담받기 |
| Product use | 15분 안에 설정하기 |
| Newsletter | 다음 글 받아보기 |

Formula:

`[행동 동사] + [받는 것/얻는 것] + [조건 or 시간]`

## Section Patterns

### Problem Section

Structure:

1. Name the specific moment.
2. Show the cost of leaving it unsolved.
3. Bridge to a new way.

Template:

```markdown
매주 [상황] 때문에 [비용/불편]이 생깁니다.
문제는 [겉으로 보이는 원인]이 아니라 [진짜 원인]입니다.
[제품/방법]은 이 과정을 [새 방식]으로 바꿉니다.
```

### Benefit Section

Structure:

1. Feature
2. So what
3. Example

Template:

```markdown
**[기능명]**
[기능 설명]. 그래서 [독자에게 생기는 변화]가 가능합니다.
예를 들어 [구체 상황]에서 [결과]를 확인할 수 있습니다.
```

### Proof Section

Use only real proof:

- Customer quote
- Before/after metric
- Case study
- Screenshot or workflow
- Third-party review
- Public benchmark

If missing:

`[증거 필요: 이 주장에는 고객 사례나 수치가 필요합니다.]`

## Social Hooks

### Pattern: Mistake

- 많은 팀이 콘텐츠 전략에서 제일 먼저 틀리는 것
- 전환율을 고치기 전에 확인해야 할 한 가지

### Pattern: Contrarian

- 키워드 볼륨만 보고 글감을 정하면 늦습니다
- 좋은 카피는 더 멋진 문장이 아니라 더 정확한 약속입니다

### Pattern: Practical List

- 랜딩페이지 첫 화면에서 바로 지워야 할 문장 5개
- 한국어 카피가 번역투처럼 보이는 이유 7가지

### Pattern: Scene

- 금요일 오후 5시, 또 블로그 주제를 못 정했다면
- 광고는 돌아가는데 문의가 없다면, 카피보다 먼저 이것을 보세요

## Product Detail Page Pattern

For Korean commerce/product pages:

1. Who this is for
2. Main before/after
3. Product proof: material, specs, process, review, guarantee
4. Use scenes
5. Objection handling
6. Purchase CTA

Avoid:

- Empty emotional claims
- Repeated superlatives
- Unverified health/legal/performance claims
- Long feature lists with no use scene

## Rewrite Moves

| Problem | Move |
|---|---|
| Too abstract | Add a scene |
| Too soft | Name the cost |
| Too salesy | Add proof or caveat |
| Too long | Split one message per paragraph |
| Too generic | Name the exact reader |
| Too cold | Add human context |
| Too casual | Restore clear terms and proof |
