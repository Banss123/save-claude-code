# Korean Style Guide

Use this reference for Korean-language editing, proofreading, localization, and tone calibration.

## Editing Passes

### 1. Meaning

Confirm the copy has a clear reader promise:

- What problem is named?
- What outcome is promised?
- What evidence supports it?
- What should the reader do next?

If the answer is unclear, fix the message before polishing wording.

### 2. Sentence Rhythm

Prefer natural Korean verb sentences over noun-heavy imported structure.

Weak:

- 고객 경험 향상을 위한 데이터 기반 의사결정 지원을 제공합니다.

Better:

- 고객 행동 데이터를 보고, 다음에 무엇을 고쳐야 할지 바로 판단할 수 있습니다.

Guidelines:

- Keep most sentences under 25 Korean words.
- Use one idea per sentence.
- Split stacked modifiers before the noun.
- Avoid repeating the same sentence ending across many lines.
- Use short standalone sentences for emphasis, but sparingly.

### 3. Translationese

Watch for English-shaped Korean:

| Pattern | Problem | Better move |
|---|---|---|
| `~를 통해` overuse | Mechanical connector | Use a direct verb |
| `~에 대한` chains | Noun stack | Rewrite as a clause |
| `~할 수 있도록 지원합니다` | Weak agency | Say who does what |
| `~에 있어` | Formal filler | Delete or simplify |
| `~하는 데 도움을 줍니다` | Vague help | Name the result |
| `~ 기반의` | Abstract | Say what the base is used for |
| `솔루션/플랫폼/시스템` | Category fog | Name the specific tool or use case |

### 4. Tone Levels

Pick one default tone:

| Tone | Use when | Watch out |
|---|---|---|
| `합니다` | Enterprise, official, expert | Can become stiff |
| `해요` | Friendly SaaS, education, creators | Can feel too casual for high-stakes categories |
| `한다` | Editorial, thesis, analysis | Can feel cold or arrogant |
| Casual fragments | Social hooks, community posts | Can hurt trust if overused |

Do not mix `합니다` and `해요` in the same asset unless quotes or UI labels require it.

### 5. Korean Marketing Slop

Flag these words when unsupported:

- 혁신적인
- 차별화된
- 독보적인
- 최적화
- 고도화
- 니즈
- 인사이트
- 시너지
- 고객 경험 향상
- 비즈니스 성장
- 데이터 기반
- 올인원
- 압도적인
- 완벽한
- 쉽고 빠르게

Replace with:

- The exact user problem
- A specific before/after
- A concrete workflow
- A proof point
- A constraint or tradeoff

### 6. Specificity

Use specificity that helps decision-making:

- Who: `1인 사업자`, `B2B SaaS 마케터`, `프랜차이즈 점주`
- Moment: `광고비가 오르는데 전환율이 떨어질 때`
- Constraint: `개발자 없이`, `30분 안에`, `월 10만 원 이하로`
- Proof: `전환율`, `시간 절감`, `고객 수`, `리뷰`, `사례`

If proof is missing, write `[증거 필요: 수치/사례/고객 인용]` rather than inventing it.

### 7. Headings

Good Korean headings should carry meaning without the body:

Weak:

- 주요 기능
- 더 나은 경험
- 왜 필요한가요?

Better:

- 매주 반복되는 보고서를 15분 안에 끝냅니다
- 신규 고객이 첫 주문까지 가는 길을 줄입니다
- 지금 놓치고 있는 검색 수요를 먼저 찾습니다

### 8. Paragraphs

For web writing:

- Keep paragraphs short: 1-3 sentences.
- Put the strongest sentence first.
- Use bullets for choices, steps, and comparisons.
- Do not make every paragraph the same length.

For essays/newsletters:

- Allow more rhythm variation.
- Use a concrete scene before abstract explanation.
- Keep one central idea per issue.

### 9. Punctuation and Formatting

- Avoid excessive exclamation marks.
- Use quotation marks only when the phrase needs emphasis or is a real quote.
- Avoid decorative emoji unless the channel expects it.
- Keep spaces around English product names and numerals readable.
- Use numerals for measurable claims: `3가지`, `15분`, `월 29,000원`.
- Keep Korean list items parallel in grammar.

### 10. Final QA

Before delivering:

- Read the first line alone. Does it earn the next line?
- Read headings only. Does the argument still make sense?
- Scan all CTAs. Are they specific?
- Check every superlative. Is it proven?
- Check every English word. Is it needed?
- Check every sentence ending. Is the tone consistent?
