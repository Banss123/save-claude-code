## 변경 포인트 (현안 → 가안)

### 1. 메뉴명에 "1인 도시락세트" 통일
- 모든 돈까스 메뉴를 "1인 도시락세트" 또는 "1인 세트" 브랜딩
- 1인 객단가 명확화 + 포장 기대값 설정

### 2. 프리미엄 원재료 강조
- "[프리미엄 한돈]" 브래킷 반복 노출
- 카테고리 헤더 "제주산 프리미엄 한돈 돈까스"

### 3. 장르 브래킷 [일식] / [분식집 감성]
- 덮밥 = [일식], 떡볶이 세트 = [분식집 감성]
- 메뉴마다 장르 태깅으로 기대치 설정

### 4. "500원 행복" 무료 서비스 대폭 확대
- 현안 12종 → 가안 11종 (유사 규모, 배치 재정렬)
- "어마왕 500원의 행복" — 브랜드화된 옵션 그룹

### 5. 카테고리 헤더 맥락화
- 분류 명사("세트메뉴", "덮밥") → 사용 맥락("같이먹어요", "필수 추가메뉴")

### 6. 옵션 그룹명을 의인화/액션어로
- "소스추가" → "푹~ 찍어먹는 소스 추가"
- "고기추가" → "오늘은 곱빼기로 [고기 추가]"
- "사이드 추가" → "한 젓가락 부족할 땐, [사이드 추가]"

## 학습 인사이트 (이 사례에서 추출)

1. **1인 도시락 업종 공식**: 메뉴명에 "1인 도시락세트/1인 세트" 접미사 통일 → 1인 객단가 기대값 제어
2. **원재료 등급 브래킷**: [프리미엄 한돈], [제주산] — 반복 노출로 프리미엄 포지션
3. **장르 브래킷 5축**: [일식] / [분식집 감성] / [오늘은 일식] — 업종 혼합 메뉴의 장르 라벨링
4. **"500원 행복" 브랜딩**: 리뷰 유도용 무료 서비스를 브랜드화된 옵션 그룹으로 운영 — 저비용 고지각가치
5. **옵션 그룹명 액션화**: 단순 분류어 → 의인화/액션어로 전환 ("푹~ 찍어먹는", "한 젓가락 부족할 땐", "오늘은 곱빼기로")
6. **고기 추가 그램 단위 세분화**: 등심/안심/치즈/고구마치즈/명란카츠 × (100g/200g) — 객단가 부스터 10단계
