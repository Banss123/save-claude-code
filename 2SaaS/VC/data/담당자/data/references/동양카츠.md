## 옵션 그룹 (가안 5개)

|---|--------|------|

### 메뉴별 옵션 매핑
- 모든 돈카츠 메뉴: `2,3,1,4,5` (전 옵션 풀 적용)
- 면류: 옵션 없음
- 사이드: 옵션 없음

→ **돈카츠 = 5옵션 풀 적용** — 카츠+카츠 추가 + 면 세트 + 사이드 + 음료 + 리뷰
→ **옵션 그룹 수가 5개로 적음** — 작은 메뉴판 특성

## 변경 포인트 (현안 → 가안 전략)

1. **카테고리에 "국내산 A++ 등급만 고집하는" 브랜드 스테이트먼트 삽입**
2. **"카츠 → 돈카츠(돈까스)" 용어 변경** — 일본어 카츠 + 한국어 돈까스 병기
3. **"1인 도시락세트" 전면 확장** — 혼밥 타겟
4. **"면류 → 돈카츠 옆에 두고 먹는 면류"** — 곁들임 프레임 카피
5. **"추가메뉴 → 곁들여 먹는 사이드메뉴"** — 역할 명확화
6. **면류 메뉴명에 감각 형용사** (따끈한/느끼함 잡아주는/시원한)
7. **치킨 가라아게 신설** — 사이드에 신메뉴 추가 (6P 9,000)
8. **옵션 "오늘은 더 든든하게, 카츠 추가"** — 친근한 톤 유도
9. **리뷰이벤트 옵션명 "hoo기 작성 el벤트"** — 플랫폼 검열 우회 (후기→hoo기, 이벤트→el벤트)
10. **가격/메뉴 구성 거의 동일** — 브랜딩 카피 집중 작업

## 학습 인사이트

1. **돈카츠 전문점은 카테고리 4개면 충분** — 돈카츠/면류/사이드/음료
2. **"국내산 A++ 등급" 같은 직설적 품질 시그널** — 카테고리 헤더에 브랜드 제1 메시지 노출
3. **"돈카츠/돈까스/카츠" 3중 병기** — 검색 키워드 최적화 관행
4. **면류는 곁들임 옵션으로 이중 배치** (카테고리 + 옵션 3번)
5. **옵션 그룹명이 대화톤** ("오늘은 더 든든하게", "돈카츠 옆에 두고 먹는")
6. **리뷰이벤트 우회 표기(hoo기/el벤트)** — 배민/쿠팡 플랫폼 검열 우회 공통 패턴
7. **2지점(본점/중동점) 동일 메뉴 시트 분리** — 지점별 탭 구조 표준
8. **[반반]/[3가지 맛]** 세트 구성 시 선호 패턴 (1+1 / 3종 세트)
9. **뽀로로 음료(밀크/딸기)** — 돈카츠 가족 고객 타겟 음료 추가
10. **모든 돈카츠 메뉴가 동일 옵션 셋(2,3,1,4,5) 적용** — 옵션 운영 일관성
