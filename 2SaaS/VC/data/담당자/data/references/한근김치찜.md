# 한근김치찜 — 메뉴판 가안 학습 노트

## 핵심 정체성 (가게 메시지)
- **"주문즉시 조리시작, 1인 김치찜"** — 즉석 조리 신뢰도
- **"신선한 재료로 만든, 반근김치찜"** — 신선함
- **"푸짐한 한상차림, 한근김치찜"** — 브랜드명 = 양(한근)
- **"어머니의 집밥이 생각나는 도시락정식"** — 감성 프레임
- **"고기에 진심인 사장님의 '고기추가'"** — 인간미 어필

## 변경 포인트 (현안 → 가안 전략)

1. **"김치찜 → 묵은지김치찜"** 전면 교체 — 숙성도 차별화
2. **"정식 → 백반도시락"** — 도시락 프레임으로 재포지셔닝
3. **카테고리에 가게 내레이션** — "주문즉시 조리시작", "어머니의 집밥이 생각나는"
4. **"사이드는 사이드 메뉴만 주문불가" 명시** — 운영 정책 카테고리로 포장
5. **"고기에 진심" 반복 브랜딩** — 사장님 인격화
6. **꽁치폭탄, 해장 맞춤, 밥도둑 야식** 등 용도/시간대 태깅
7. **ZI뷰** = 리뷰의 검열 우회 표기 (ㄹ→ZI) → 플랫폼 필터 회피
8. **옵션 14개 단계적 확장** (현안 12개 → 가안 14개, 재료 선택을 독립 그룹으로)
9. **기존 태그(_야식,안주) 제거** — 메뉴명 깔끔하게
10. **닭발은 별도 카테고리 운영** — 김치찜 전문점의 야식 부수 카테고리

## 학습 인사이트

1. **브랜드명이 양(한근) = 카테고리 명(반근/한근)** — 브랜드명 자체가 분량 시그널
2. **김치찜 → 묵은지김치찜 용어 교체는 고숙도 프리미엄 신호**
3. **"정식" 대신 "백반도시락"** — 동일 메뉴 재명명으로 배달/도시락 채널 최적화
4. **재료 변주(돼지/삼겹/우삼겹/스팸/꽁치) + 양 변주(반근/한근)** 매트릭스 구조
5. **사장님 인격화 카피("고기에 진심인 사장님의")** — 신뢰도 어필
6. **ZI뷰/el벤트 같은 검열 우회 표기** — 플랫폼 정책 회피 패턴 (다른 매장도 공통)
7. **옵션 그룹이 14개로 대규모** — 김치찜은 변주가 많은 업종
8. **리뷰 선물을 옵션으로 정교하게 분리** (ZI뷰 + 리뷰이벤트 2중)
9. **사이드 구성이 밥/국물/튀김/전/고기류 풀세트** — 한식 상차림 완성도
