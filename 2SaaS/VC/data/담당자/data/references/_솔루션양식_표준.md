## 핵심 결론 — "유연한 컴포넌트 모델"

5건 비교 결과 단일 고정 양식 아님. **표지·본문·꼬리의 모든 컴포넌트가 옵션**이며 매장별로 조합이 다름. 양식은 2가지 변종으로 분기:

| 변종 | 케이스 | 특징 |
|------|--------|------|
| **A. 전통형** | 담솥(9p), 종가밀면(7p) | 매출 KPI 3~4박스, Before/After 4~5행, 섹션 7~8개, 사장님 확인 + 수수료 포함, 항목 섹션별 번호 |
| **B. 진단형** | 강제로떡볶이(5p), 부엉이돈까스(7p), 갓꾸운닭(6p) | 진단 KPI 4박스(CTR/CVR/CPC/객단가), 지금/앞으로 3행, 섹션 4~5개, 항목 직렬 번호, 사장님 확인/수수료 생략 |

## 고정 요소 (모든 양식 공통)

1. **제목**: "솔루션 계획서" (+옵션: "(YY.MM.DD.요일 기준)" 작성일자)
2. **부제**: "[업장명] | 배달[앱] 매출 최적화 컨설팅" (+옵션: 위치/업종 부가)
3. **핵심 지표 박스 영역** (1줄, 2-6개 박스)
4. **비교표 영역** (Before/After 또는 지금/앞으로)
5. **본문 번호 섹션** (① ② ... 형식)
6. **표지 끝 안내문**: "※ 아래 세부 계획을 검토하신 후, 조정이 필요한 부분을 말씀해주시면 반영하겠습니다."

## 가변 요소 (매장별 다름)

| 컴포넌트 | 가변 범위 | 비고 |
|----------|-----------|------|
| 페이지 번호 헤더 ("N/M") | 있음/없음 | 부엉이만 매 페이지 헤더 |
| 작성일자 | 있음/없음 | 강제로떡볶이/부엉이/갓꾸운닭 = 있음 |
| 핵심 지표 박스 개수 | 2~6개 | 매장 핵심 진단 지표 자유 조합 |
| 핵심 지표 박스 종류 | (a) 매출 (현재/+3%/+5%) (b) 객단가 (c) CTR/CVR/지역평균 (d) CPC/예산/CMG/최소주문 | 자유 조합 |
| 비교표 헤더 라벨 | "Before/After" 또는 "지금/앞으로" | 변종별 |
| 비교표 행 수 | 3~5행 | 메뉴판/광고/운영 (필수) + 썸네일/플랫폼 (옵션) |
| 비교표 끝 인용문 | 있음/없음 | 컨설턴트 핵심 코멘트 (회색 박스) |
| 섹션 수 | 4~8개 | 매장 상황별 |
| 섹션 번호 표기 | ① 원형 + 진녹색 헤더 / ① 원형 + 회색 박스 / 단순 텍스트 | 변종별 |
| 항목 번호 매김 | 섹션별 (1.1, 1.2...) / 직렬 (1, 2, ... N) | 변종별 |
| 강조 마커 | ★ (별, 핵심) / ▸ (삼각형, 일반) / – (대시, 하위) / 빨간 텍스트 (경고) | 자유 |
| 회색/녹색 인용 박스 | 섹션 내 자유 삽입 | 컨설턴트 코멘트 |
| 본문 데이터 표 | 옵션 | 키워드 검색량, 채널별 입점 매트릭스 등 |
| 사장님 확인 섹션 (1~10) | 있음/없음 | 전통형만 |
| 수수료 안내 섹션 | 있음/없음 | 전통형만 (1차/2차 후불제) |
| 푸터 회사명 | 있음/없음 | "ValueChain | 배달 플랫폼 매출 최적화 컨설팅" |

## 새 SolutionPlan 스키마 설계 (Pydantic 모델 명세)

```python
class StoreInfo(BaseModel):
    name: str  # "담솥 청주점"
    business_type: str | None = None  # "한식·도시락"
    location: str | None = None  # "청주"
    document_date: str | None = None  # "26.03.18.화" (옵션)

class CoreMetricBox(BaseModel):
    """핵심 지표 박스 (1개)."""
    label: str  # "계약서 작성 기준 월매출" / "현재 CTR" / "배민 CPC 단가 (예정)"
    value: str  # "1,100만원" / "2.5%" / "388원"
    sub_label: str | None = None  # 보조 설명
    is_target: bool = False  # 목표값 (녹색 강조)
    is_baseline: bool = False  # 기준값 (오렌지 강조)

class ComparisonRow(BaseModel):
    """비교표 행 (1개)."""
    category: str  # "메뉴판" / "썸네일" / "광고" / "운영" / "플랫폼"
    before_lines: list[str]  # 여러 줄 가능
    after_lines: list[str]  # 여러 줄 가능
    after_first_bold: bool = True  # 첫 줄 볼드 강조

class ComparisonTable(BaseModel):
    """비교표 전체."""
    header_label: Literal["Before/After", "지금/앞으로"] = "지금/앞으로"
    rows: list[ComparisonRow]  # 3~5행
    footer_quote: str | None = None  # 회색 박스 컨설턴트 코멘트

class SectionItem(BaseModel):
    """섹션 내 항목 (1개)."""
    number: str | None = None  # "1" (직렬 번호) 또는 "1.1" (섹션별) 또는 None
    title: str  # "사장님추천 6개 구성"
    marker: Literal["star", "arrow", "warning", "none"] = "arrow"  # ★/▸/빨간경고/없음
    bullets: list[str] = []  # ▸ 또는 – 항목들
    sub_descriptions: list[str] = []  # – 추가 설명
    quote_box: str | None = None  # 회색/녹색 인용 박스
    data_table: dict | None = None  # 옵션 데이터 표

class Section(BaseModel):
    """본문 섹션 (1개)."""
    number: int  # 1, 2, ... (자동 ① ② ...)
    title: str  # "배민 기본 세팅"
    items: list[SectionItem]
    footer_quote: str | None = None  # 섹션 끝 코멘트

class OwnerCheckItem(BaseModel):
    """⑥ 사장님 확인 항목 (1개)."""
    number: int  # 1~10
    description: str

class FeeTier(BaseModel):
    label: str  # "1차 목표 매출"
    amount: str  # "1,800만원"
    rate_pct: int  # 3

class FeeStructure(BaseModel):
    tiers: list[FeeTier]
    notes: list[str] = ["후불제: 매출 목표 달성 시에만 과금", "목표 매출 미달 시 수수료 없음"]

class SolutionPlan(BaseModel):
    """ValueChain 솔루션 계획서 (유연 모델)."""
    # 표지
    store: StoreInfo
    document_title: str = "솔루션 계획서"
    subtitle_suffix: str = "배달앱 매출 최적화 컨설팅"
    core_metrics: list[CoreMetricBox] = Field(min_length=2, max_length=6)
    comparison: ComparisonTable
    cover_footer_note: str = "※ 아래 세부 계획을 검토하신 후, 조정이 필요한 부분을 말씀해주시면 반영하겠습니다."

    # 본문
    sections: list[Section] = Field(min_length=3)
    item_numbering: Literal["per_section", "serial"] = "serial"  # 항목 번호 방식

    # 꼬리 (모두 옵션)
    owner_checks: list[OwnerCheckItem] | None = None  # ⑥ 있음/없음
    fee_structure: FeeStructure | None = None  # ⑦ 있음/없음

    # 메타
    company_name: str = "ValueChain"
    show_page_numbers: bool = False  # "N/M" 헤더
    document_date: str | None = None  # 작성일자
```

## DocxBuilder v2 책임 (코드 고정)

1. **표지 레이아웃**:
   - 제목 (큰 폰트) + 작성일자 옵션
   - 부제 ("[업장명] | 배달앱 매출 최적화 컨설팅")
   - 핵심 지표 박스 N개 가로 배치 (1개 행, N열, 회색 테두리)
   - 비교표 (헤더 진녹색 + 카테고리 좌측 연녹색 + after 첫 줄 볼드)
   - 끝 안내문 (이탤릭 회색)

2. **섹션**:
   - 섹션 헤더: ⓪ + "제목" + 진녹색 배경 1행 또는 회색 박스
   - 항목: 마커 (★/▸/–) + 제목 + bullets + sub_descriptions + 인용 박스
   - 항목 번호 직렬 vs 섹션별 (option)

3. **꼬리**:
   - 사장님 확인 (옵션): ⑥ 섹션 + (1)~(N) 번호 리스트
   - 수수료 안내 (옵션): ⑦ 섹션 + 표 + ▸ 후불제 안내
   - 푸터: "ValueChain | 배달 플랫폼 매출 최적화 컨설팅" + "본 계획서는..." 인용

4. **색상 (코드 고정)**:
   - 진녹색: `#1E3A32` (섹션 헤더, Before/After 헤더)
   - 연녹색: `#E8F2EC` (비교표 카테고리, 핵심지표 박스)
   - 오렌지: `#F49E33` (기준값 강조 — 매출 등)
   - 녹색: `#2B7A4B` (목표값 강조, 수치 강조)
   - 회색: `#666666` (안내문, 인용 박스 텍스트)
   - 빨간색: `#E60000` (경고 텍스트)
   - 회색 박스 배경: `#F2F2F2`
   - 녹색 박스 배경: `#E8F2EC`

## 학습 인사이트 (5건 비교 통합)

1. **양식 자체가 유연**: 컨설턴트가 매장 상황에 맞춰 컴포넌트 자유 조합. 고정 템플릿 X
2. **변종 A (전통형)** = 매출 목표 위주, 풀스펙 (사장님확인+수수료 포함)
3. **변종 B (진단형)** = 진단 지표 위주, 간소형 (사장님확인+수수료 생략)
4. **AI 판단 시 변종 선택 기준**:
   - 매출 데이터 풍부 + 정식 계약 단계 → A형
   - 진단 지표 데이터 풍부 + 신속한 액션 플랜 단계 → B형
5. **공통 디자인 시그널**: 진녹색 헤더, 연녹색 강조, 마커 ★/▸/–, 회색/녹색 인용박스
6. **회사명 ValueChain은 푸터 노출** (담솥/종가밀면) 또는 생략 (B형 매장)
7. **본문 데이터 표** (키워드 검색량, 채널별 입점 매트릭스)는 부엉이처럼 컨텐츠 풍부할 때 추가
