## 2026-04-15 — 리서치 산출물 부풀림

### 실패 모드
사용자가 "카테고리별 상위 3곳 서치해보면 안 되나?"라고 가볍게 제안한 것을 거대 리서치 작업으로 부풀려 15개 카테고리 DB(`docs/업종별_기준.md`, 334줄)를 완성품처럼 적재했음. 하지만 산출물이 사용자 기준("메뉴판을 맛있게 세팅한 잘하는 매장")에 미달했고 → 전량 폐기 지시.

### 감지 신호
- 사용자의 가벼운 제안 톤 ("~해보면 안 되나?") vs 내가 만든 결과물 규모
- "잘한 곳" 정의를 사용자에게 사전 합의 없이 내가 임의 해석 (= 프랜차이즈 상위 = 잘함, 잘못된 통념)
- 산출물 1건도 사용자 검증 받기 전에 15개로 확장

### 예방 규칙
1. **사용자 가벼운 제안 → 1건 시범 후 확장 여부 재확인**. 처음부터 풀스케일 작업 금지.
2. **리서치/큐레이션 전 "좋음의 정의" 사용자에게 1문장으로 합의**. "프랜차이즈 상위 = 잘함"처럼 통념을 자체 가정하지 말 것.
3. **공식 기준 데이터/DB는 사용 사례 1건 검증 후에만 확장**. "데이터 모았으니 쓰일 것"은 금물.
4. **서브에이전트 발사 = 큰 결정**. 병렬 발사 직전에 "이 결과로 무엇이 어떻게 좋아지는가" 한 줄 자문.
