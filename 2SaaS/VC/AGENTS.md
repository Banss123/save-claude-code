## 핵심 원칙 (요약)

상세는 위 파일들 참조. 잊지 말 것:

- **사용자가 최종 의사결정자.** 독단 행동 금지, 큰 결정·destructive git·secret 처리는 컨펌 필수.
- **정확성 최우선.** 결정론 파이프라인이라 산출물이 틀리면 안 됨. 추측 금지.
- **검증 없이 완료 보고 금지.** `uv run pytest` + `uv run ruff check .` 통과 후 보고.
- **수치 출처 없으면 쓰지 마라.** 통계·비율·평균값 추측 작성 절대 금지.
- **새 의존성 추가 신중히.** 기존 스택으로 해결 가능하면 추가하지 않음.
- **실제 고객 데이터·계정/세션·API key는 커밋하지 않음.**
