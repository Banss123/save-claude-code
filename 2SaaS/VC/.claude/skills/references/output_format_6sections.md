# 솔루션 계획서 6섹션 — 정본 §출력 형식 매핑

`src/generator/docx_builder.py` 의 `_build_lever_report_section()` (L1105 이하) 가 정본 `data/references/목표매출_산정로직.md` §출력 형식(문서 라인 199-303) 의 6섹션 양식을 1:1 구현한다. `document-build` 스킬이 이 파일을 단일 진실 출처(SoT)로 참조한다.

## 정본 ↔ 코드 매핑

| 정본 §섹션 | 코드 함수 |
|-----------|----------|
| 섹션 1. 헤더 (매장명/카테고리/산정일) | `_lr_section1_header()` |
| §출력 1. 레버별 현황 분석 | `_lr_section2_levers()` |
| §출력 2. 목표매출 산정 | `_lr_section3_targets()` |
| §출력 3. 희망매출 괴리 분석 | `_lr_section4_hope_gap()` |
| §출력 4. 달성 선행 조건 | `_lr_section5_prerequisites()` |
| §출력 5. 산정 한계/가정 | `_lr_section6_assumptions()` |

## 활성화 조건

6섹션 레이아웃은 `lever_report is not None` 일 때만 발동. `None` 이면 기존 tier 표 폴백.

## 검수 체크리스트

DOCX 의 "목표매출 산정서" 블록이 아래 6섹션을 **순서대로** 포함하는지 확인:

- [ ] §1. 헤더 (매장명 / 카테고리 / 산정일 / 현재 매출 배민·쿠팡·요기요·합계)
- [ ] §2. 레버별 현황 분석 (노출/CTR/CVR/객단가 4개 레버 × 단기/중기 개선폭 + basis)
- [ ] §3. 목표매출 산정 (1차 3% 구간 / 2차 5% 구간 / 플랫폼별 + 합계 + 확률 + 예상 수수료)
- [ ] §4. 사장님 희망매출 괴리 분석 (희망치 입력 시만, 없으면 스킵)
- [ ] §5. 달성 선행 조건 (매장주 실행 필수 항목 + 데이터 추가 수집 필요)
- [ ] §6. 산정 한계/가정 명시 (벤치마크·상권·계절성·실행 준수 가정)
