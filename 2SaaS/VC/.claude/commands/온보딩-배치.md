`value-chain-onboarding` 에이전트에게 아래 작업을 위임합니다.

**대상 매장 목록**: $ARGUMENTS

위 매장들을 쉼표로 분리해서 **순차로** 온보딩 파이프라인을 실행해 주세요.

실행 지침:
1. 매장명을 콤마 기준으로 파싱합니다(앞뒤 공백 제거).
2. 각 매장마다 `onboarding-full` 스킬을 한 번씩 호출합니다. **병렬 실행 금지** — 배민 세션이 공유되어 충돌 위험이 있습니다.
3. 한 매장이 실패해도 다음 매장으로 진행하되, 실패한 매장은 실패 사유와 함께 별도 목록으로 기록합니다.
4. 매장마다 주요 검수 결과(passed/failed)를 누적 보고합니다.
5. 모든 매장 처리 후 최종 요약을 제시합니다:
   - 성공 매장 수 / 실패 매장 수
   - 매장별 생성 파일 경로
   - 검수 FAIL이 있는 매장 목록과 주요 실패 항목

참고:
- 배치 API(50% 할인)는 현재 파이프라인이 Claude LLM 호출을 하지 않으므로 적용 대상이 아닙니다(규칙 기반 결정론).
- 대량 처리 시에는 매장당 평균 5-10분을 예상하세요.

절대 금지:
- `src/**`, `pyproject.toml`, `tests/**` 수정 금지
- JSON 내용 직접 편집 금지
- 병렬 실행 금지 (세션 충돌)
- `git commit` 금지
