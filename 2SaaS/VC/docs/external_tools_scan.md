## 2. 카테고리별 요약

- **매출 분석**: 배민 셀프서비스(원천), 앳트래커(홀+배달 통합), Bloom(엔터프라이즈 LTV), Toast/Olo(미국 표준). 국내 SMB용 배달 전용 통합 분석은 빈약.
- **메뉴 최적화**: Popmenu(웹+마케팅), MenuTiger(QR 메뉴). **배달 가격·옵션 구조 최적화 전용 SaaS는 공개 시장에서 미발견** → 핵심 차별점.
- **리뷰 자동화**: 바리뷰·리프리·딜리봇·앳트래커(국내 레드오션, 월 1~3만원대), Ovation·Marqii·Bloom(해외 엔터프라이즈).
- **배달앱 파트너 전용**: 배민 셀프서비스만 존재. 광고·고객 데이터는 있으나 **목표매출 레버 산정 기능 없음**.
- **오픈소스**: piabona(한국어 리뷰 AI), BG-NBD LTV, XGBoost 수요예측. 모델 레퍼런스로 유용.

## 3. 우리 프로젝트와의 갭

**우리가 이미 제공**: 4-레버 목표매출 정본 로직, 배민 스크래핑+메뉴판 가안 XLSX(셀 단위 볼드/한글 폰트 고정), 6섹션 솔루션 DOCX, 담당자 룰 기반 결정론 생성(AI는 판단만).

**그들이 있고 우리는 없는 기능**:
1. 이탈·LTV 예측 (Bloom, BG-NBD) — 레버 보정에 접목 여지
2. 리뷰 AI 답변 (바리뷰·piabona) — 현재 범위 밖
3. SEO·리스팅 일관성 (Marqii) — 네이버 플레이스 확장 시
4. POS 데이터 통합 (Toast·앳트래커) — 홀 매출 합산 시

**차별점**: 배민 특화 end-to-end, 담당자 룰 YAML, 4-레버 곱셈 정본(공개 경쟁자 미발견), XLSX+DOCX 동시 생성.

## 4. 즉시 도입·참고 후보

1. **piabona/AI-reply-generator** — 한국어 리뷰 데이터셋·프롬프트 재사용. 통합 난이도: 낮음 (MIT 가정, 파인튜닝 결과물만 참조).
2. **BG-NBD + Gamma-Gamma (mukulsinghal)** — 과거 주문 데이터가 있으면 고객 LTV/리피트 확률을 레버 보정에 투입. 난이도: 중 (pandas+lifetimes 패키지).
3. **Ovation "2-clicks AI 제안" UX** — 리뷰뿐 아니라 솔루션 수정 제안에도 적용 가능. 난이도: 낮음 (UI 패턴 차용).
4. **Bloom의 "22개 데이터소스 통합" 컨셉** — 배민+쿠팡이츠+요기요+땡겨요 통합 스키마 설계 시 레퍼런스. 난이도: 중.
5. **앳트래커 "홀+배달 통합 매출"** — 담당자가 POS 데이터도 요구하면 우선 벤치마크. 난이도: 중.

## 5. 결론

**외부 툴 대체 불가** — 우리 핵심(4-레버 목표매출 + 메뉴판 가안 XLSX + 솔루션 DOCX 자동 생성)은 공개 시장에서 동등 제품을 찾지 못함. 국내는 "리뷰 답변" 한 기능에 4개 업체가 집중되어 있고, 해외는 POS·마케팅·리스팅 통합이지 **컨설팅 산출물 생성**이 아님.

**자체 구축 가치** — 있음. 단, 리뷰 답변·LTV·이탈 예측은 오픈소스·레퍼런스를 차용하면 재개발 비용 절감. 우리는 "배민 수정 가능한 컨설팅 오토메이션"이라는 포지션을 유지하고, 주변 모듈은 외부 벤치마크를 흡수하는 전략이 합리적.
