# BizHigh SalesOps 시연 시나리오

> 목적: 피드백 미팅에서 "이 도구가 매일 열어볼 만한가"를 빠르게 검증한다.
> UI 취향 토론보다 실제 업무 흐름을 먼저 보여준다.

## 0. 준비

```bash
npm run dev
```

- 로컬 접속: `http://localhost:3000/app`
- 프로덕션 테스트: `https://banss-salesops-vercel.vercel.app`
- 내 정보에서 현재 사용자 선택: 김민재 또는 반민성
- Supabase 초기화가 필요하면 `supabase db reset`

## 1. 대시보드 첫 화면

보여줄 포인트:
- 담당자 필터로 내 매장만 좁혀진다.
- "누락"은 연체 + 관리 점검 지연을 합친 위험 신호다.
- 헤더 알림 종에 pending 알림 count가 뜬다.

멘트:
- "오늘 들어오면 먼저 이 숫자와 알림만 보면 됩니다."

## 2. 지금 가장 급한 1건

기본으로 선택되는 퀘스트 또는 `분당 미소치과 - 4주치 아티클 주제 컨펌 받기`를 클릭한다.

보여줄 포인트:
- 왼쪽: 매장 상태, 회차, 키워드, 바로가기 링크
- 가운데: 업주 성향, 선호 채널, 민감 포인트
- 오른쪽: 지금 해야 할 액션, 다음 퀘스트, 완료/위임/스킵/메모

멘트:
- "퀘스트 하나만 눌러도 이 매장 기억을 다시 불러오게 하는 게 핵심입니다."

## 3. 실제 액션

`메모만` 또는 `위임`을 먼저 누른다.

보여줄 포인트:
- 메모는 quest metadata에 쌓인다.
- 위임은 `assignee_id`에 저장된다.
- 완료는 `quest_completions`를 만들고 트리거가 quest status와 activity_log를 처리한다.

주의:
- 실제 시연에서 `완료`를 누르면 해당 퀘스트가 보드에서 사라지고 완료 모음으로 이동한다.
- 되돌리기 기능은 있지만, 시연에서는 `메모만` 또는 `위임`을 먼저 보여주는 게 안전하다.

## 4. 체크리스트 누락 퀘스트

`여의도 한방내과 - 체크리스트 미기입 확인`을 찾는다.

보여줄 포인트:
- source가 `sheet_missing`인 항목은 완료 버튼이 아니라 `체크리스트 ↗` 버튼이 뜬다.
- 이 항목은 SaaS에서 처리하는 게 아니라 시트에서 처리해야 한다.

멘트:
- "시트를 source of truth로 두되, 놓친 항목만 SaaS가 끌어올립니다."

## 5. 알림 종

헤더의 종 아이콘을 누른다.

보여줄 포인트:
- `sheet_missing`, `medical_law_pending`, `health_stale` 같은 운영 알림이 한 곳에 모인다.
- 알림의 `맥락 열기`를 누르면 대시보드의 해당 퀘스트 컨텍스트로 이동한다.

## 6. 간트차트

대시보드 하단 간트차트를 보여준다.

보여줄 포인트:
- 위 칸은 내 작업, 아래 칸은 업주 소통 활동 강도다.
- 퀘스트·이벤트·시작일이 한 달 흐름에 같이 보인다.
