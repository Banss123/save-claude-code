# 매장 관리 프로세스 (도메인 지식)

> 비즈하이가 매장을 받아 관리하는 표준 프로세스. SaaS의 핵심 = 이 프로세스를 **자동으로 퀘스트로 풀어내** 누락 방지하는 것.

## 협업 모델

이 SaaS는 **3명**이 사용. 분담 없이 **셋이 같이 보고 같이 처리**. 모든 매장·퀘스트는 세 사람 모두에게 보임 (필터로 "내 담당만" 옵션은 가능).

| 역할 | 인물 | 주로 하는 일 (참고, 강제 분담 X) |
|---|---|---|
| **영업자** | 김민재 / 김재원 | 외부 영업·계약·입금·업주 소통·미팅·클레임 |
| **인하우스 마케터** | 반민성 | 컨텐츠·자료·매장 세팅·소식글·아티클·리뷰 작업자 |

각 단계의 "담당" 컬럼은 **주로 누가 하는지 참고**일 뿐, 누가 처리해도 됨. 퀘스트 큐는 셋이 공유.

## 사용 도구 매핑

| 도구 | 용도 |
|---|---|
| **카카오워크** | 본사 내부 협업 (워크방·팀 채널) — 견적 컨펌, 시작일 요청, 주간 보고서 포워딩 |
| **카카오톡** | 업주 직접 소통 (계약서 전자서명 발송, 시작일 안내, 보고 등) |
| **구글시트** | 견적서 양식, 계약 양식, 히스토리, 매장 체크리스트, 리뷰 체크리스트 |
| **구글폼** | 신규 DB 구글폼 (온보딩 자료 기반 수동 기입) |

## 큰 흐름

```
매장 등록 (영업자가 신규 매장 추가)
   ↓
[A. 계약 전 프로세스] — 입금 확인까지
   ↓
[B. 계약 후 프로세스] — 시작일 확정·세팅까지
   ↓
[C. 관리 중 프로세스] — 주기적 반복 업무
```

매장 등록 시점에 SaaS가 위 단계들을 **퀘스트(=할 일)로 자동 생성**, 우선순위 큐에 적재. 사용자(영업자/마케터)는 메인 대시보드에서 우선순위 순으로 처리하고 "완료" 클릭 → 다음 단계 등장.

## A. 계약 전 프로세스

| # | 단계 | 산출물·조건 | 담당 |
|---|---|---|---|
| 1 | 업주 "계약할게요" 의사 표명 → 정보 수집 | 개월수 / 키워드수 / 매장명 / 단가 / 할인율 / **결제수단(카드)** | 영업자 |
| 2 | 견적서 엑셀 작성 (필수: 개월수·금액·매장명) → 사진 추출 | 견적서 이미지 | 영업자 |
| 3 | 본사 내부 컨펌 + 온보딩 자료 전달 받기 | 컨펌 완료 | 영업자 |
| 4 | 업주에게 견적서·온보딩 자료 전송 | — | 영업자 |
| 5 | 사업자등록증 / 이메일 수취 | 파일·이메일 보관 | 업주 → 영업자 |
| 6 | 계약서 요청 (전자서명: 내가 서명하면 업주에게 카카오톡 자동 전달) | 전자서명 발송 완료 | 영업자 |
| 7 | 입금 대기 → 입금 확인 시 내부 입금확인 양식 기록 | 입금확인 기록 | 영업자 |

**확정 사항** (2026-04-26):
- 1번 결제수단(카드) — 매장 등록 시 함께 메모/세팅하는 필드.
- 7번 입금확인 — 사용자가 내부 양식에 **수동 작성**. SaaS는 "입금확인 완료" 체크리스트로만 추적 (자동 발송 X).
- 6번 전자서명 서비스 — TBD (실제 사용 중인 서비스명 확인 필요, SaaS 연동 여부도 함께).
- 기본 키워드 수는 3개. 추가 키워드 월 가산 단가는 요식업·뷰티 10만원, 병의원·약국 20만원 (VAT 별도).
- 기본 월 공급가(VAT 별도): 요식업·뷰티 1개월 130만원 / 3개월 110만원 / 6개월 95만원 / 12개월 80만원. 병의원·약국 1개월 150만원 / 3개월 130만원 / 6개월 115만원 / 12개월 100만원.
- 매장 등록 폼은 업종·키워드 수·계약기간으로 월 공급가를 자동 계산하되, 실제 견적이 다를 수 있어 수동 단가와 할인 후 단가를 별도로 입력할 수 있음.

## B. 계약 후 프로세스

| # | 단계 | 비고 | 담당 |
|---|---|---|---|
| 0 | 영업 사전 작업 (계약 전 단계와 병행 또는 직후) | 맞춤제안서 제작 (키워드 추출·순위 분석), 광고 소재 편집, 메타광고 세팅 (설명문·타겟·인스턴트 양식) | 마케터 |
| 1 | 신규DB 구글폼 작성 | **온보딩 자료 기반 수동 기입** (자동 동기화 없음) | 영업자 |
| 2 | 업주 팀채팅방 개설 | 기본 채널: 카카오톡 팀채팅방 | 영업자 |
| 3 | ↑ 채팅방에 초대: 영업자 / 업주 / 전무님 / 인하우스 마케터 | — | 영업자 |
| 4 | 구글비즈니스프로필 관리자 권한 요청 | GBP 표준 권한 요청 흐름. 세팅 필요 매장은 **GBP 프로필 세팅** 퀘스트로 별도 추적 | 영업자 → 업주 |
| 4a | **GBP 비즈니스 인증 완료 체킹** | 인증 안 되면 다음 진행 불가 | 영업자 |
| 5 | 매장 자료 요청 (사진·메뉴·정보·기타 + **업주가 어필하고 싶은 사항**) | 업종별 분기 발생 | 영업자 → 업주 |

| 5c | 사진 자료 드라이브 생성 + 매장 자료 전달 양식 작성 → 매장 워크방 전달 | — | 마케터 |
| 6 | 리뷰 작업자 구인 | — | 영업자/마케터 |
| 7 | 본사에 리뷰 작업자 인적사항 보고 | — | 영업자 |
| 8 | 리뷰 작업자 톡방 개설 | — | 영업자 |
| 9 | 시작일 요청·확정 (**수동 입력**) | 5b 컨펌 + 7·8 완료가 선행. ⭐ 시작일 = 후속 모든 일정의 anchor | 영업자 |
| 10 | 업주에게 시작일 포워딩 | — | 영업자 |
| 11 | 작업자 리뷰원고 / 답글양식 컨펌 | — | 마케터/상무 |

**확정 사항** (2026-04-26):
- 1번 신규DB 구글폼 — 영업자가 온보딩 자료 보고 **수동 기입**. SaaS↔폼 자동 동기화 없음.
- 9번 시작일 — 영업자가 **수동 입력**. 입력 즉시 C 단계의 모든 주기 작업(주간/중간등수/월간 보고, 매장·리뷰 체크리스트) 일자가 자동 산출되고 퀘스트가 자동 생성·알림.
- 2번 팀채팅방 — 통합 채널 TBD (현재는 외부 도구 사용, SaaS 통합 여부는 추후).

## C. 관리 중 프로세스 (주기 반복)

매장이 "관리 시작" 상태가 되면 다음이 주기적으로 발생. **모든 일자는 시작일(B9) 기준 자동 산출** → 퀘스트 자동 생성 + 알림.

### C-1. 시작일 후 1회성 마일스톤

| 항목 | 시점 | 담당 | 비고 |
|---|---|---|---|
| **D+15일 업주 연락** ⭐ | 시작일+15일 | 영업자 | 첫 안부·진행 상황 공유 |
| 업주에게 프로필 세팅 정보 수취 | 시작 직후 | 영업자 → 업주 | 구글 프로필 보강용 |

### C-2. 정기 보고

| 항목 | 주기 (시작일 기준) | 산출물 | 담당 |
|---|---|---|---|
| 주간보고 | 매주 | 보고서 (본사 컨펌 후 송부) → 워크방 포워딩 | 마케터 |
| 중간등수 보고 | 주기 TBD (월 1회 또는 2주) | 키워드 순위 스크린샷·요약 (컨펌 후 송부) | 마케터 |
| 월간보고 | 매월 (시작일+30·60·90일) | 종합 리포트 + **월간보고 미팅** + 미팅 후 송부 | 마케터·영업자 |
| 아티클 컨펌 (병의원·약국 한정) | 4주마다 | 다음 4주 아티클 (컨펌 후 송부) | 업주 ↔ 마케터 |

### C-3. 정기 체크리스트

| 항목 | 주기 | 내용 | 담당 |
|---|---|---|---|
| 매장 체크리스트 | 주기 TBD (구글시트) | GBP·매장 정보·소식글 발행 상태 등 | 마케터 |
| 리뷰 체크리스트 | 주기 TBD (구글시트) | 리뷰 작업·답글 점검 | 마케터 (작업자 리딩) |
| 영업자 소식글 체크 | 주기 TBD | 본사가 발행한 소식글 적절성 점검 | 영업자 |

### C-4. 이벤트성 / 비정기

| 항목 | 트리거 | 담당 |
|---|---|---|
| 이슈 발생 시 GBP 프로필 인증 (동영상) | 인증 만료·정지 등 | 영업자 (동영상 담당) |
| 이슈 발생 시 GBP 프로필 인증 (메일) | 동일 | 마케터 (메일 담당) |
| 클레임 대응 | 업주 클레임 발생 | 영업자 |
| 인디비 리텐션 | 신규 DB 후 리텐션 | 영업자 |
| 샤오홍슈 체험단·기자단 일정 체크 | 샤오홍슈 서비스 계약 시 본사 일정양식 수취 | 마케터·영업자 |

**확정 사항** (2026-04-26):
- 일자 산출 = **시작일 anchor 기반 자동**.
- **퀘스트 적재는 1개월 단위 롤링**. 시작일 입력시 첫 1개월치만 적재 → 그 1개월 종료 시점에 다음 1개월치 자동 갱신. 6개월 한꺼번에 X (계약 변경시 일괄 수정 부담 줄임).
- 알림은 due_date 임박 시 메인 대시보드 퀘스트 우선순위 상위로 자동 노출.

**미해결**: 중간등수보고·매장체크리스트·리뷰체크리스트 정확한 주기·양식.

## 매장 상태 머신 (프로세스 기반 재정의)

`store_status` enum 후보:

```
contract_pending      # A 단계 진행 중
contract_signed       # A 완료, B 진행 중
ready_to_start        # B 완료, 시작일 대기
active                # C 진행 중 (정상 관리)
paused                # 일시 중단
churned               # 이탈
archived              # 보존
```

각 단계 진입·이탈 시 audit log 자동 기록.

## 누락 방지 매커니즘 (이 프로세스 기반)

1. **퀘스트 큐 자동 생성**: 매장 등록 = A1 퀘스트 자동 발급. A1 완료 = A2 자동 발급. 단계 누락 불가능 구조.
2. **블로커 표시**: B5b처럼 선행 조건 있는 단계는 "차단됨" 상태로 표시 (체크박스 회색).
3. **시작일 anchor + 롤링 적재**: B9 시작일 입력 = C 단계 첫 1개월치 퀘스트 생성·적재. 1개월 종료 시 다음 1개월치 자동 갱신. due_date 임박 시 자동으로 메인 우선순위 상위 노출.
4. **stale 감지**: 단계가 N일 이상 진행 안 되면 알림 (N은 단계별 다름 — TBD).
5. **퀘스트 = 단일 진실 원천**: 카톡/이메일/시트/메모에 흩어진 업무를 SaaS 퀘스트로 통합.
