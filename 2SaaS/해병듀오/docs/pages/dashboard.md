## ⭐ 간트차트 (메인 뷰)

### 의도

영업자·마케터가 대시보드 한 번 열어서 **이번 달(또는 한 차수) 모든 매장이 어디까지 와 있는지 즉시 파악**.

### 구조 (1차 안)

```
                    │ 월 │ 화 │ 수 │ 목 │ 금 │ 토 │ 일 │ ... 한 달
─────────────────────┼────┼────┼────┼────┼────┼────┼────┼─────
강남 한우다이닝 [A]  │■■■■│    │    │    │    │    │    │  계약전 (사등본 수취)
성수 카페로스터리[B] │    │■■■■■■■■│    │    │    │  온보딩 (GBP 권한)
분당 미소치과   [B*] │■■■■■■■■■■■■■■│ (블로커: 원고 컨펌)
송파 우리약국   [C] │■│  │■│  │■│■■■■■│ ... 시작일·주간보고
이태원 비스트로 [C] │   │■│  │■│  │■│  │ ... 주간보고 + 월말 월간보고
─────────────────────┴────┴────┴────┴────┴────┴────┴────┴─────
                       ↑오늘
```

- **세로**: 매장 (status별 그룹 또는 단순 정렬)
- **가로**: 시간축 (디폴트 = 한 달, 토글로 한 차수 = 계약 기간)
- **막대**: 진행 중 단계 색상 + 마일스톤 점 + due 임박 빨강
- **블로커**: B.5b 같은 차단 단계는 빗금/경고 아이콘
- **오늘 라인**: 세로 빨강 선
- **클릭**: 매장 상세 → 또는 해당 퀘스트 활성

### "한 차수" 정의

= 한 매장의 한 계약 기간(예: 6개월). 매장별로 차수 다름. 토글: 월간 / 차수 단위.

### 보조 영역 (간트 아래 또는 우측)

영상 시안 차용:
- **인사 메시지**: "{이름}님, 오늘도 퀘스트 N건 남았습니다"
- **상단 통계 4개**: 관리매장 / 진행중 퀘스트 / 오늘마감 / 연체
- **활동 히트맵** (26주, 두 줄: 내 작업 / 업주 소통)
- **퀘스트 보드** (탭: 전체/핀고정/긴급/오늘마감/완료)
- **글로벌 검색 + 알림**

### 미해결

- 매장 50개+가 되면 세로 스크롤 길어짐. 그루핑·필터 디자인?
- 간트 막대 안에 어떤 정보 노출? (단계 코드만? 텍스트?)
- 퀘스트 보드와 간트차트 동시 노출 vs 탭 전환?

## 사이드바 (전 페이지 공통)

- 대시보드
- 매장등록
- 매장관리
- DB 관리(`/app/leads`)

정기체크·보고서 페이지는 남아 있지만 사용자 결정상 메뉴 우선순위에서는 후순위다.

## 퀘스트 영역

### 데이터 모델

```
quest:
  id              uuid
  store_id        uuid (FK)         # 어느 매장의 일인지
  title           text              # "사업자등록증 수취" 등
  description     text              # 상세
  process_step    text              # process.md의 단계 ID (예: "A.5")
  priority        int               # 정렬용 (낮을수록 위)
  status          enum              # pending / completed / blocked
  blocked_by      uuid[]            # 선행 퀘스트 ID 리스트
  due_date        date              # 마감일 (선택)
  created_at      timestamptz
  completed_at    timestamptz
  source          enum              # auto (프로세스 자동 생성) / manual (사용자 추가)
```

### 표시 규칙

- **진행 중 영역**: `status = pending` AND `not blocked` → priority ASC, due_date ASC 정렬
- **완료 모음**: `status = completed` → completed_at DESC (최근 완료가 위)
- **blocked 항목**: 회색 처리해서 표시 (왜 차단됐는지 hover로 표시)

### 인터랙션

- 카드 클릭 → 상세(매장 링크, 다음 액션 가이드) 펼침
- "완료" 버튼 → `status = completed`, `completed_at = now()` → 메인에서 사라지고 완료 모음으로 이동
- 완료시 다음 단계 퀘스트 자동 활성 (process.md 단계 의존성 따라)

### 퀘스트 출처

1. **자동** (대다수): 매장 등록 → process.md A1 퀘스트 자동 생성. A1 완료 → A2 자동 생성. (단계 누락 불가능)
2. **수동** (보조): 사용자가 직접 추가 (예: "B매장 사장님 통화하기")

## 캘린더

- **기본 뷰**: 월간 (전체 흐름 파악용)
- **표시 항목**:
  - 매장별 진행 마일스톤 (예: "OO매장 시작일", "XX매장 월간보고 마감")
  - 미팅 일정 (영업·마케터·업주 미팅)
- **클릭** → 해당 일정 상세 / 관련 매장으로 이동

## 미해결 항목 (확인 필요)

- 퀘스트 상세 카드 안에 들어갈 정보 (매장명/단계/마감/액션 버튼 등)는 어떻게?
- 캘린더 이벤트 입력은 어디서? (캘린더 직접 클릭 / 매장 상세 / 미팅 별도 페이지?)
- 미팅과 매장 진행상황을 캘린더에서 시각적으로 구분? (색깔/아이콘?)
- 퀘스트 큐가 너무 많아질 때 (예: 50개 이상) 그루핑/필터(매장별·우선순위별) 필요한가?
- **사용자 표명**: "대시보드는 간트차트 형식" (출처: docs/sources/02_큰그림_프로세스.txt). 현재 카드 리스트 + 캘린더와 다름. 통합/분리 결정 필요.
