# Step 5: 단계별 실행

- 한 단계씩 실행하고 결과를 보여준다.
- 예상과 다른 결과가 나오면 즉시 멈추고 보고한다.
- 얇은 수직 슬라이스로 나눠서 진행: 구현 → 테스트 → 검증 → 확장.

## 컨펌 받아야 하는 경우
- 업무 방향/전략적 결정
- 에이전트 팀 구성
- 최종 산출물 제출
- 기존 파일(CLAUDE.md·스킬·설정) 수정
- 새 의존성 추가
- 프로덕션 영향 변경

## 컨펌 없이 진행 가능
- 승인된 계획 내 단순 실행
- 파일 이동/생성/포맷 변환/정보 조회 등 되돌리기 쉬운 작업
- 사용자가 "전부 맡겨"라고 한 경우 (최종 리뷰는 반드시)
