# 다국어·다매장 콘텐츠 사전 체크리스트

> 다국어 콘텐츠 작업(리뷰 답변 / GBP 소식글 / 뉴스레터 / 상세페이지 / 소셜 포스트 등) 시작 전, 클로드가 먼저 자가 선언한 뒤 사용자 승인 받고 작성한다. 사후 교정이 아니라 사전 차단이 목적.

---

## 사전 자가 선언 (5줄 + 승인)

콘텐츠 생성 작업 첫 응답에 아래 5줄 블록을 먼저 출력하고, 사용자 "ㅇㅇ" / "ㄱ" / "맞아" 받은 뒤 작성 시작.

```
1. 작업 언어: [원문과 동일 / 한국어 출력 등 명시]
2. 포함할 SEO 키워드: store-info.md에서 추출 → [키워드1, 키워드2, ...]
3. 보존할 표현: 사용자가 강조한 셀링 포인트(독점·한정·공식·1호점 등)
4. 중국어인 경우: 간체 / 번체 (원본 자료 변형 기준)
5. 출력 경로: [저장 위치]
```

생략 조건:
- 단축어 "그냥 해" / "전부 맡겨" → 자가 선언 후 즉시 작성 (승인 대기 X)
- 같은 세션에서 동일 매장·동일 작업 반복 → 첫 회만 선언, 이후 변경점만 짧게 명시

---

## 언어 매칭 절대 룰

- **원문 언어 = 답변 언어**. 터키어 리뷰 → 터키어 답변. 일본어 리뷰 → 일본어 답변. 중국어 리뷰 → 중국어 답변.
- **모르는 언어를 영어로 fallback 절대 금지**. 차라리 "이 언어 작성 자신 없음, 검증 부탁드립니다" 명시.
- 한국어 답변이 정책인 매장은 store-info.md에 명시돼 있어야 적용. 명시 없으면 원문 언어 기본.

### 체크 시점
- 리뷰 답변: 리뷰어별로 언어 다를 수 있음 → 매 리뷰마다 언어 식별 후 답변 언어 결정
- 소식글: 매장 운영 정책 (KR 단일 / KR+CN / KR+JP+CN 등) 매장별 다름 → store-info.md 확인

---

## SEO 키워드 누락 금지

- **첫 초안부터 store-info.md SEO 키워드 포함**. 사용자가 "키워드 빠졌네"라고 지적해야 들어가는 흐름 절대 금지.
- 매장명·시술명·지역명·카테고리명 등 store-info.md에 정의된 키워드를 모두 자연스럽게 배치.
- 적용이 어려운 콘텐츠 형태(예: 짧은 리뷰 답변)면 그 자리에서 "이 키워드는 자연스럽게 못 들어가는데, 빼도 될까요?"로 협의.

### 중국어 변형 (간체 / 번체)
- store-info.md에 간체/번체 명시돼 있으면 그대로 따름.
- 명시 없으면 **원본 자료(PDF·이미지·기존 콘텐츠) 변형 기준** 매칭.
- 추측으로 변환 절대 금지. 모르면 사용자에게 확인.

---

## 핵심 표현 보존

- 사용자가 한 번이라도 강조한 셀링 포인트(예: 독점·국내 최초·정품 보장·1호점·VIP 전용)는 편집·다듬기 과정에서 **절대 임의 삭제 금지**.
- 다듬는 게 어색해 보여도 표현 자체를 빼지 말고, 위치만 옮기거나 톤만 부드럽게 조정.
- 명백한 사실 오류·법적 리스크가 있을 때만 사용자에게 먼저 알리고 합의 후 수정.

---

## 출력 포맷 일관성

- 같은 작업 묶음 내 모든 언어가 **동일한 헤더·인사·서명 구조**를 공유. (예: 한·중·일 리뷰 답변에서 인사말 위치, 매장명 표기, 서명이 통일돼야 함)
- 리뷰 답변은 리뷰어 이름 헤더 + 스킬 파일에 정의된 표준 인사 필수.
- 폴더 빈 상태 보고 전 `ls` + `Glob`/`Read` 이중 검증.

---

## 위반 시 회복 절차

콘텐츠 송출 전 사용자가 위반을 발견한 경우:
1. 어디서 어긋났는지 한 줄로 자기 진단 (어느 룰의 어느 항목)
2. 이 룰의 어느 부분을 강화해야 재발 안 하는지 1줄 제안
3. 동의 시 이 파일 또는 관련 스킬 파일 즉시 업데이트 (Step 6 즉시 반영 원칙)
