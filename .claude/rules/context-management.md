# 컨텍스트 관리 (세션을 깨끗하게)

- **쓰기 전에 읽어라**: 편집 전에 권위 있는 소스(기존 모듈/패턴/테스트)를 찾아라.
- **작업 시작 시 자동 탐색**: 업무 시작 전에 관련 파일 경로들을 읽고 현재 상태를 파악. 별도 카탈로그 문서 없이, 매번 실제 파일을 직접 읽어서 최신 상태 기준으로 작업.
- **작업 노트 유지**: `tasks/todo.md`에 핵심 제약, 불변량, 결정, 발견된 함정을 기록.
- **컨텍스트가 커지면**: 간략한 요약으로 압축하고 잡음은 버려라.
- **범위 확장 통제**: 더 깊은 이슈 발견 시 정확성/안전에 필요한 것만 고치고 나머지는 TODO로 남겨라.
- **컨텍스트 오염 방지**: 대량 데이터(파일 전문, API 응답, 검색 결과)는 메인에 직접 쏟지 말고 서브에이전트로 처리하거나 핵심만 추출해서 보고.
- **대화가 길어지면**: 현재 작업 상태를 요약 정리하고 불필요한 중간 과정은 버려라.
- **단계 전환 시 압축**: 리서치→계획, 계획→구현, 디버깅 완료→다음 작업 넘어갈 때 중간 과정 정리. 결론/결정/파일 경로만 남기고 시행착오는 버려라.
- **실패 후 접근 변경 시**: 기존 삽질 히스토리를 정리하고 새 접근으로 전환.
