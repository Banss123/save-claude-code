# Step 7: Reflect — 주간 회고

작업 단위 회고가 아닌 **7일치 횡단 회고**. 일주일에 한 번 자동/수동 트리거.

## 트리거
- **자동**: 일요일 밤 (cron 또는 schedule 스킬 활용 — 추후)
- **수동**: `/회고` 또는 "주간 회고" 자연어 요청
- **임시**: 큰 마일스톤 종료 후

## 데이터 소스

| 소스 | 활용 |
|---|---|
| 노션 로그 DB (`4c86dd0a-4d34-41b6-a19f-c6b5c037258f`) | 한 주의 모든 대화/결정/피드백 |
| `~/.claude/projects/.../memory/MEMORY.md` | 새로 추가된 메모리 |
| 6개 프로젝트 `tasks/todo.md` (있는 경우) | 완료/미완료 항목 |
| 프로젝트 폴더 변경 | git log 또는 파일 mtime |

## 산출물 (노션 페이지)

자동 생성된 회고 페이지에 다음 섹션 채움:

1. **잘 풀린 패턴 3개 + 그 이유** (성공 분석)
2. **막힌 지점 + 근본 원인** (실패 분석)
3. **다음 주 우선순위 후보** (자동 추출)
4. **새 규칙 후보** (반복된 피드백 → CLAUDE.md/rules 반영 제안)
5. **메모리 갱신 제안** (auto memory에 추가/수정할 항목)

## 호출 방법
스킬 `~/.claude/skills/skills/weekly-reflect/SKILL.md` 사용.

## 다음 단계
- 회고 후 → 사용자 검토 → 규칙/메모리 갱신 승인 받기
- 승인된 사항 즉시 반영 (Step 6의 "즉시 반영 원칙")
- 다음 주 사이클 시작
